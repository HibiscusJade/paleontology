# 中国古生物学会官方网站 — 建设进展与需求

---

## 一、项目概述

### 1.1 建设目标

中国古生物学会官方网站（含11个专业分会）旨在实现以下核心目标：

1. **会员管理规范化**：建立统一的会员信息库，实现入会、缴费、续费全流程线上化，每笔缴费凭证与发票对应，满足财务审计要求。
2. **会议服务一体化**：覆盖学术会议从发布、报名、缴费、摘要征集的完整业务闭环。
3. **内容发布自主化**：建成后可自行维护全站内容，无需依赖技术人员。
4. **党建展示规范化**：为党建文化专栏提供独立的内容管理与展示体系。

### 1.2 建设原则

| 原则 | 说明 |
|------|------|
| 统一会员体系 | 只设一种会员身份——中国古生物学会会员，统一收取会费，不再按分会单独收取 |
| 单账号全站通用 | 一个邮箱对应一个账号，适用于全部11个分会 |
| 分会自主绑定 | 会员根据学术兴趣自主选择加入分会，可多选，可随时变更 |
| 权限隔离 | 分会内部会议及专属内容仅对已加入该分会的有效会员开放 |
| 单笔单次缴费 | 会员费统一缴纳一次；会议费按每场会议独立缴纳，确保财务对账清晰 |
| 两阶段财务审核 | 凭证券审 → 发票终审，两个环节独立操作，凭证与发票一一对应 |
| 全流程可追溯 | 每笔缴费、每次审核均记录时间与操作人，满足审计整改要求 |

### 1.3 当前建设阶段

```
阶段一：需求梳理与交互原型验证  已完成
阶段二：后端系统与后台管理开发  待启动
阶段三：测试部署与内容迁移     后续
```

当前已完成**可交互原型系统**，完整覆盖会员管理、会议管理、分会管理等全部核心业务流程。原型可直接在浏览器中操作验证，实现了全部26个页面的页面结构、功能流转与数据交互。

## 二、系统功能架构

### 2.1 网站栏目结构

网站分为**学会主站**与**党建文化专栏**两大板块，共含11个一级导航栏目：

**学会主站（10个栏目）：**

| 序号 | 一级栏目 | 功能定位 |
|------|---------|---------|
| 1 | 首页 | 新闻动态、通知公告、近期会议概览、分会入口导航 |
| 2 | 学会简介 | 学会概况、章程、历届领导、理事会、监事会、秘书处 |
| 3 | 组织机构 | 11个专业分会总览、总会组织架构 |
| 4 | 学会服务 | **核心业务模块**：会员入会、会费缴纳、会议报名、摘要提交、科学传播、科技奖励 |
| 5 | 学会沿革 | 学会发展历程时间线 |
| 6 | 历史相册 | 早期风采、学术会议、野外考察、国际交流等影像资料 |
| 7 | 会员公告 | 学术会议、奖励申报、组织工作、科普教育等分类公告 |
| 8 | 国际交流 | 交流动态、国际会议、合作机构 |
| 9 | 资料下载 | 管理办法、学术标准、年报资料、会员表格等分类文件 |
| 10 | 规章条例 | 学会章程、保护条例、国际准则 |

**党建文化专栏（12个子栏目）：**

| 序号 | 子栏目 | 功能定位 |
|------|--------|---------|
| 1 | 通知公告 | 党内通知发布 |
| 2 | 党群机构 | 党群组织架构展示 |
| 3 | 党委纪委 | 党委、纪委信息发布 |
| 4 | 党建工作 | 党建工作动态与成果展示 |
| 5 | 组织生活 | "三会一课"等组织活动记录 |
| 6 | 党员队伍建设 | 发展党员、教育管理等信息 |
| 7 | 理论学习专栏 | 学习资料、心得体会 |
| 8 | 党建专题 | 专题活动归集（如党纪学习教育） |
| 9 | 先进典型 | 优秀党员、先进组织表彰 |
| 10 | 违纪举报 | 监督举报入口 |
| 11 | 下载中心 | 党建文件集中下载 |

> **注：** 党建专栏在网站上拥有独立的导航区域与视觉呈现，与学会主站形成清晰的功能区隔。

## 三、核心业务系统设计

### 3.1 会员管理体系

#### 3.1.1 业务逻辑总览

学会采用**统一会员制**：用户注册后，缴纳一次会费，经审核后成为"中国古生物学会有效会员"，有效期一年。有效会员可自主选择加入任意专业分会，接收分会通知、参加分会会议。

会员资格的获取与分会绑定完全解耦——会费统一缴纳给学会，分会仅作为学术归属，不涉及独立收费。

#### 3.1.2 入会缴费流程（两阶段审核）

根据学会财务审计要求，会费审核分为两个独立阶段：

**阶段一：凭证初审**

| 步骤 | 操作主体 | 操作内容 |
|------|---------|---------|
| 1 | 申请人 | 确认会员类型（普通会员/学生会员/单位会员），确认对应会费标准 |
| 2 | 申请人 | 通过银行柜台或网银向学会对公账户转账 |
| 3 | 申请人 | 上传银行转账回执（支持JPG/PNG格式，≤5MB） |
| 4 | 财务审核员 | 核验凭证真实性、金额准确性，做出初审决定 |

- **初审通过** → 申请人获得"可绑定分会"权限，同时开启发票上传窗口（7个工作日时限）
- **初审驳回** → 注明驳回原因，申请人重新上传凭证

**阶段二：发票终审**

| 步骤 | 操作主体 | 操作内容 |
|------|---------|---------|
| 5 | 申请人 | 在凭证初审通过后7个工作日内，上传电子发票（支持JPG/PNG/PDF格式，≤10MB） |
| 6 | 系统 | 自动识别发票金额，与凭证金额进行比对 |
| 7 | 财务审核员 | 审查发票真实性及金额比对结果，做出终审决定 |

- **终审通过** → 正式成为有效会员，有效期自审核通过之日起1年
- **终审驳回** → 注明驳回原因，申请人重新上传发票

**逾期处理机制：**

| 情形 | 处理规则 |
|------|---------|
| 超过7个工作日未上传发票 | 会员资格自动锁定，已选分会保留但功能暂停 |
| 逾期后补传发票 | 上传后正常进入终审流程，通过后自动解锁 |
| 距截止日≤3天 | 系统自动发送提醒通知 |
| 特殊情况需延期 | 财务审核员可在后台为指定申请人手动延长截止日期 |

#### 3.1.3 会员状态定义

系统将会员划分为9种状态，每种状态对应明确的业务含义与操作权限：

| 状态 | 含义 | 后续操作 |
|------|------|---------|
| 未入会 | 已注册账号，尚未申请入会 | 进入入会流程 |
| 凭证初审中 | 凭证已提交，等待财务确认 | 预计1-3个工作日内完成 |
| 凭证被驳回 | 凭证审核未通过 | 查看驳回原因，重新提交凭证 |
| 待上传发票 | 凭证初审已通过 | 7个工作日内上传发票；此阶段可先行绑定分会 |
| 发票逾期 | 未在规定时间内上传发票 | 补传发票即可恢复正常流程 |
| 发票终审中 | 发票已提交，等待财务最终确认 | 预计1-3个工作日内完成 |
| 发票被驳回 | 发票审核未通过 | 查看驳回原因，重新提交发票 |
| 正式会员 | 审核全部通过，会员资格生效 | 有效期1年；可绑定分会、报名会议 |
| 已过期 | 会员有效期届满 | 续费后恢复会员资格 |

#### 3.1.4 分会绑定机制

- **绑定条件：** 仅有效会员（凭证初审已通过及以上状态）可进行分会绑定操作
- **绑定数量：** 不设上限，可同时绑定0~11个分会
- **生效方式：** 即时生效，无需审核
- **变更自由：** 可随时绑定或解除，系统不设限制
- **到期处理：** 会员到期后，所有分会绑定自动解除；续费后需重新选择（考虑会员的学术兴趣可能发生变化，不做自动恢复）

#### 3.1.5 会员到期后的关联业务处理

当会员资格到期时，系统对已报名的会议资格采取分级处理：

| 会议报名状态 | 处理方式 | 业务依据 |
|------------|---------|---------|
| 已确认 (终审通过) | **资格保留** | 已完成全部审核流程，不应追溯取消 |
| 待终审 (发票已提交) | **资格冻结** | 流程接近完成，给予续费恢复机会 |
| 其他中间状态 | **自动取消** | 流程处于早期阶段，续费后重新报名更合理 |

### 3.2 会议管理体系

#### 3.2.1 业务逻辑总览

各分会发布学术会议后，系统仅向**已加入该分会且为有效会员**的用户推送。用户需独立完成每场会议的报名缴费流程——同一用户可以同时报名多场会议，每场会议的缴费与审核独立进行，互不影响。

#### 3.2.2 会议报名缴费流程（两阶段审核）

会议费的审核流程与会费一致，采用两阶段设计：

**阶段一：凭证初审**

| 步骤 | 操作内容 |
|------|---------|
| 选择参会身份 | 学生/教师/嘉宾，系统自动计算对应注册费标准 |
| 银行转账 | 根据页面所示的收款账户信息完成转账 |
| 上传凭证 | 上传转账回执（JPG/PNG，≤5MB） |
| 财务初审 | 财务审核员核验凭证 |

**初审通过后，用户获得两项权限：**
- 填写参会信息（姓名、单位、报告类型、住宿选择、摘要上传）
- 上传电子发票（7个工作日时限）

**阶段二：发票终审**

| 步骤 | 操作内容 |
|------|---------|
| 上传发票 | 上传电子发票（JPG/PNG/PDF，≤10MB），系统自动进行金额比对 |
| 财务终审 | 财务审核员综合审查后做出终审决定 |

- **终审通过** → 报名确认，获得正式参会资格
- **终审驳回** → 注明原因，重新上传发票

#### 3.2.3 会议报名状态定义

| 状态 | 含义 | 参会信息填写 | 发票上传 |
|------|------|------------|---------|
| 未缴费 | 尚未开始报名流程 | 不可操作 | 不可操作 |
| 凭证初审中 | 凭证已提交，等待审核 | 不可操作 | 不可操作 |
| 凭证被驳回 | 凭证审核未通过 | 不可操作 | 不可操作 |
| 待上传发票 | 凭证初审已通过 | **可填写** | **需上传（含截止日提醒）** |
| 发票逾期 | 超过上传期限 | 已锁定（保留已填数据） | 开放补传 |
| 发票终审中 | 发票已提交，等待终审 | 只读 | 不可操作 |
| 发票被驳回 | 发票审核未通过 | 只读 | 重新开放上传 |
| 已确认 | 全部审核通过 | 只读 | 不可操作 |

#### 3.2.4 参会信息管理

**填写内容：**

| 信息项 | 说明 |
|--------|------|
| 基本信息 | 姓名、性别、单位——系统自动读取，无需重复填写 |
| 报告类型 | 口头报告 / 展板报告 / 仅参会 |
| 报告标题 | 选择口头报告或展板报告时需填写 |
| 摘要文件 | 上传Word格式摘要（.doc/.docx，≤10MB） |
| 住宿选择 | 单间 / 双人间 / 自行安排 |

**管理规则：**
- 参会信息及摘要在会议设定的截止日期前可反复修改，系统仅保留最新版本
- 截止日期后，所有信息锁定为只读状态
- 后台管理员保留在特殊情况下强制修改的权限

#### 3.2.5 会议生命周期

每场会议经历以下四个阶段：

| 阶段 | 特征 | 用户侧表现 |
|------|------|----------|
| 预告期 | 会议已发布，报名尚未开放 | 可浏览会议详情，报名按钮不可用 |
| 报名期 | 报名开放，未到缴费截止日 | 可正常报名缴费 |
| 已截止 | 缴费截止日已过 | 不可报名；已报名者可继续完成后续流程 |
| 已结束 | 会议日期已过 | 会议归档至历史记录 |

### 3.3 通知体系

系统通过站内消息与邮件两种渠道，对全部关键业务节点进行自动通知：

| 通知节点 | 通知对象 | 渠道 |
|---------|---------|------|
| 凭证初审通过/驳回 | 申请人 | 站内信 + 邮件 |
| 发票终审通过/驳回 | 申请人 | 站内信 + 邮件 |
| 发票上传截止日临近（距截止日≤3天） | 申请人 | 站内信 + 邮件 |
| 发票已逾期 | 申请人 | 站内信 + 邮件 |
| 新会议发布 | 已加入对应分会的有效会员 | 站内信 + 邮件 |
| 缴费截止日临近 | 已报名未缴费者 | 站内信 + 邮件 |
| 会员资格即将到期 | 会员本人 | 站内信 + 邮件 |
| 会员到期致会议资格冻结/取消 | 会员本人 | 站内信 + 邮件 |
| 参会信息/摘要提交成功 | 申请人 | 站内信反馈 |

## 四、后台管理系统（下一阶段开发范围）

### 4.1 内容管理系统

支持学会自主维护全站11个一级栏目及党建12个子栏目的全部内容：

- 富文本编辑器（图文排版、表格、视频嵌入）
- 新闻/公告/动态的发布、编辑、置顶、下架
- 学会简介中的人员信息（领导、理事、秘书处成员）的结构化管理
- 历史相册的图片分类与批量管理
- 资料下载中心的文件分类与权限设置
- 党建专栏各子栏目的独立内容维护入口
- 每个分会独立的内容管理空间（分会管理员仅可操作本分会内容）

### 4.2 财务审核工作台

- **双队列管理：** 初审队列（凭证审核）与终审队列（发票审核）独立展示与操作
- **凭证/发票预览：** 支持原图在线查看，OCR识别结果辅助对照
- **批量处理：** 支持多选和全选后进行批量审核通过/驳回
- **驳回原因：** 驳回时强制填写原因，通知自动发送给申请人
- **手动延期：** 针对特殊情况，可为指定缴费记录延长发票上传截止日
- **操作日志：** 每次审核操作记录操作人、时间、IP地址、操作详情

### 4.3 权限体系

| 角色 | 人数 | 权限范围 |
|------|------|---------|
| 学会总管理员 | 约5人 | 全站内容管理、全部功能数据查看与操作、系统配置 |
| 分会管理员 | 每分会约2人 | 本分会页面内容编辑、本分会会议管理、查看本分会会员统计 |
| 财务审核员 | 若干 | 会费与会议费的凭证/发票审核（对其他功能模块仅可浏览） |

### 4.4 数据统计与导出

**统计面板：**

| 统计维度 | 指标举例 |
|---------|---------|
| 会员统计 | 在籍会员总数、新增/续费趋势、按身份类别分布 |
| 分会统计 | 各分会绑定人数、绑定分布趋势 |
| 会议统计 | 各场会议报名人数、报告类型分布、住宿统计 |
| 财务统计 | 会费/会议费实收金额、审核通过率、逾期率、平均审核时长 |

**数据导出：**
- 会员名册（含分会绑定信息）
- 各分会绑定会员明细
- 缴费记录明细（含凭证与发票对应关系）
- 参会人员名册（含全部参会信息字段及摘要文件索引）
