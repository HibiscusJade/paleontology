### 1.1 技术栈

| 技术 | 版本/方案 |
|------|----------|
| 前端框架 | React 19 |
| 构建工具 | Vite 7 |
| 类型系统 | TypeScript 5.6 |

### 1.2 项目定位

当前原型是一个 **纯前端 SPA**，无后端 API。所有数据通过 `localStorage` 模拟持久化，所有交互（注册、缴费、审核、绑定）均在前端模拟完成。目标是**验证交互流程**，为后续后端开发提供参考。

## 4. 会员体系（核心）

> **这是整个系统最核心的业务逻辑，所有交互都围绕会员状态展开。**

### 4.1 全局用户状态模型

一个用户从注册到完整参与学术会议，经历以下状态层级：

```
用户状态层级（从底层到顶层）:

  ┌─────────────────────────────────────────────────────────┐
  │ 第3层：会议参与状态                                       │
  │   conferenceRegs[confId].status                          │
  │   unpaid → pending → approved_unfilled → submitted       │
  ├─────────────────────────────────────────────────────────┤
  │ 第2层：分会绑定状态                                       │
  │   boundBranches: string[]                                │
  │   条件：societyMembership.status === "active"             │
  ├─────────────────────────────────────────────────────────┤
  │ 第1层：学会会员状态                                       │
  │   societyMembership.status                               │
  │   not_member → pending → active → expired                │
  │                  ↓                                       │
  │               rejected → (重新上传) → pending             │
  ├─────────────────────────────────────────────────────────┤
  │ 第0层：账号状态                                           │
  │   isLoggedIn: boolean                                    │
  │   currentUser: User | null                               │
  └─────────────────────────────────────────────────────────┘
```

**层级依赖关系：**
- 第3层依赖第2层 + 第1层（参加会议需：会员有效 + 已绑定该分会）
- 第2层依赖第1层（绑定分会需：会员有效）
- 第1层依赖第0层（申请入会需：已登录）

#### 数据模型定义

```typescript
// ===== 用户 (User) =====
interface User {
  email: string;           // 全局唯一，不可修改
  name: string;            // 实名
  gender: "男" | "女";
  unit: string;            // 工作/学习单位
  role: "学生" | "教师" | "嘉宾";
  title?: string;          // 职称（可选）
  memberType?: "普通会员" | "外籍会员" | "单位会员" | "荣誉会员";  // 预留扩展
}

// ===== 学会会员状态 (SocietyMembership) =====
interface SocietyMembership {
  status: "not_member" | "pending" | "active" | "expired" | "rejected";
  amount?: number;             // 缴费金额
  expiryDate?: string;         // 有效期截止日期
  rejectReason?: string;       // 驳回原因
  history: PaymentRecord[];    // 历史缴费记录
}

// ===== 缴费记录 (PaymentRecord) =====
interface PaymentRecord {
  id: string;
  type: "society_fee" | "conference_fee";
  targetName: string;          // "中国古生物学会会员费" 或 会议标题
  amount: number;
  voucherUrl: string;          // 转账凭证URL
  invoiceUrl: string;          // 发票URL
  submitTime: string;          // ISO时间
  auditTime?: string;
  status: "pending" | "approved" | "rejected";
  rejectReason?: string;
}

// ===== 会议注册 (ConferenceReg) =====
interface ConferenceReg {
  status: "unpaid" | "pending" | "approved_unfilled" | "submitted";
  // 表单字段（approved_unfilled 后填写）
  name: string;
  gender: "男" | "女";
  unit: string;
  role: "学生" | "教师" | "嘉宾";
  accommodation: "单间" | "双人间" | "自行安排";
  session: string;               // 分会场
  presentationType: "口头报告" | "展板报告" | "仅参会";
  reportTitle?: string;
  abstractFileName?: string;
  lastUpdated?: string;
}

// ===== 通知 (SystemNotification) =====
interface SystemNotification {
  id: string;
  title: string;
  content: string;
  time: string;                // "YYYY-MM-DD HH:mm"
  read: boolean;
  type: "info" | "success" | "warning";
}
```

### 4.2 账号注册

#### 4.2.1 注册入口（3个触发路径）

```
路径A：顶部导航栏 "登录" 按钮
  → 弹窗打开，初始 activeTab = "login"
  → 用户手动切换到 "注册" 标签

路径B：顶部导航栏 "加入会员" 按钮（未登录状态）
  → 弹窗打开，初始 activeTab = "register"
  → 直接进入注册表单

路径C：学会服务页 → 会员服务Tab → 未登录引导区
  → 显示 "请先登录" 占位
  → "登录账号" / "注册账号" 两个按钮
  → 点击注册 → 弹窗打开，activeTab = "register"

判断逻辑（PartyLayout.tsx 第199-215行）:
  if (!isLoggedIn || societyMembership.status !== "active")
    → 显示 "加入会员" 按钮
  if (!isLoggedIn)
    → 按钮onClick → setDialogOpenTab("register"); setDialogOpen(true)
  else (已登录但非会员)
    → window.location.href = "/services?tab=member"
```

#### 4.2.2 注册表单字段与校验

```
┌──────────────────────────────────────────┐
│          注册账号（LoginJoinDialog）       │
├──────────────────────────────────────────┤
│                                          │
│  邮箱 *          [_______________]       │  ← 全局唯一校验
│  密码 *          [_______________]       │  ← ≥6位
│  确认密码 *      [_______________]       │  ← 与密码一致
│  姓名 *          [_______________]       │  ← 实名
│  性别 *          [男] [女]              │
│  单位 *          [_______________]       │
│  身份 *          [学生/教师/其他] 下拉    │
│  职称            [_______________]       │  ← 可选
│                                          │
│           [ 注册 ]                       │
│                                          │
└──────────────────────────────────────────┘
```

**校验规则（LoginJoinDialog.tsx handleRegisterSubmit）：**

| 字段 | 规则 | 错误提示 |
|------|------|---------|
| 所有带*字段 | 非空 | "请填写所有必填字段（带有 * 标记）！" |
| 密码 vs 确认密码 | 必须一致 | "两次输入的密码不一致！" |
| 密码长度 | ≥ 6位 | "密码长度不能少于6位！" |
| 邮箱 | 全局唯一（查 `MOCK_USER_DB`） | "该邮箱已被注册！" |

**注册成功后：**
1. 用户数据写入 `localStorage` (`paleo_user_db`)
2. 自动登录（设置 `currentUser`）
3. 关闭弹窗
4. toast "注册成功！欢迎加入中国古生物学会。"
5. 添加通知：`{ title: "注册成功", content: "欢迎加入...", type: "info" }`

#### 4.2.3 注册后初始状态

```
注册完成后的默认状态:
  isLoggedIn          = true
  societyMembership   = { status: "not_member", history: [] }
  boundBranches       = []
  conferenceRegs      = {}
  notifications       = [注册成功通知]
```

### 4.4 统一会费缴纳

> **核心规则：只设一种会员身份——中国古生物学会会员。统一收取一次会员费（¥200/年），不再按分会单独收费。**

#### 4.4.1 完整状态机（含所有触发条件）

```
                    ┌─────────────┐
        自动注册后   │  not_member │  ← societyMembership 初始状态
      ┌───────────→ │             │
      │             └──────┬──────┘
      │                    │ applySocietyMembership()
      │                    │ (上传凭证+发票，提交审核)
      │                    ▼
      │             ┌─────────────┐
      │     ┌───────│   pending   │  ← 等待财务审核
      │     │       └──────┬──────┘
      │     │              │
      │     │    ┌─────────┼──────────┐
      │     │    │                    │
      │     │    │ simApproveSociety  │ simRejectSociety
      │     │    │ Membership()       │ Membership(reason)
      │     │    │                    │
      │     │    ▼                    ▼
      │     │ ┌──────────┐    ┌──────────────┐
      │     │ │  active  │    │  rejected    │
      │     │ │          │    │ (含驳回原因)  │

      │     │ │ expired  │          │
      │     │ │          │          │

### 4.5 分会绑定机制

> **核心规则：有效会员可自由绑定/解绑任意分会（0~11个），绑定后立即生效，无需审核。**

#### 4.5.1 绑定权限决策树

```

解绑操作:
  用户点击 [解绑分会] 按钮

    → 注意：已报名该分会会议的状态不受影响（会议记录独立保存）
