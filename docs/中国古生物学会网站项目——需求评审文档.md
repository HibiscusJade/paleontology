## 1. 项目概述

### 1.1 项目背景

中国古生物学会及其下属 **11个专业分会** 需建设统一的 **新网站及会员管理系统**，重点解决：

- **会员管理线上化**：注册、会费缴纳、资格管理的全流程线上化
- **学术会议全流程管理**：从会议发布到参会报名的完整闭环
- **党建文化规范化展示**：满足审计整改要求
- **财务对账清晰可追溯**：凭证+发票双重审核，支撑审计

### 1.2 功能模块总览

```
┌─────────────────────────────────────────────────────────┐
│                    中国古生物学会网站                      │
├─────────────┬───────────┬───────────┬──────────────────┤
│ 模块一      │ 模块二    │ 模块三    │ 模块四            │
│ 会员管理    │ 会议管理  │ CMS内容   │ 分会管理          │
│             │           │ 管理      │ （11个子站点）    │
├─────────────┼───────────┼───────────┼──────────────────┤
│ 模块五      │ 模块六：系统支撑                              │
│ 党建文化    │ 消息通知 / 文件管理 / 数据统计 / 权限系统      │
└─────────────┴─────────────────────────────────────────────┘
```

### 1.3 核心设计原则

| 原则 | 说明 |
|------|------|
| **单账号全局通用** | 一个邮箱一个账号，全站所有分会通用 |
| **权限隔离** | 分会间的会员/会议数据严格隔离 |
| **单次缴费** | 会员费、会议费均单笔单次缴纳 |
| **智能+人工审核** | 凭证和发票先OCR识别，再人工审核 |
| **最新版本原则** | 参会信息和摘要只保留最后一次更新版本 |
| **自动通知** | 所有关键操作自动触发站内通知+邮件通知 |

## 2. 模块一：会员管理模块

> **核心模块**：管理学会会员的完整生命周期：注册 → 申请加入分会 → 缴费 → 审核 → 续费 → 过期

### 2.1 功能清单

| 功能编号 | 功能名称 | 用户端 | 管理端 | 优先级 |
|---------|---------|--------|--------|--------|
| M1-01 | 会员注册 | ✅ 邮箱注册（姓名/性别/单位/身份） | — | P0 |
| M1-02 | 会员登录 | ✅ 邮箱+密码登录 | — | P0 |
| M1-03 | 密码找回 | ✅ 邮箱找回密码 | — | P0 |
| M1-04 | 个人信息管理 | ✅ 修改除邮箱外的个人信息 | — | P0 |
| M1-05 | 分会申请与加入 | ✅ 选择分会→申请加入 | — | P0 |
| M1-06 | 会员费缴纳（含凭证+发票） | ✅ 线下转账→上传凭证→上传发票 | ✅ 审核 | P0 |
| M1-07 | 会员状态查看 | ✅ "我的分会"展示各分会状态 | ✅ 会员查询 | P0 |
| M1-08 | 会员费续费 | ✅ 到期前可续费 | — | P0 |
| M1-09 | 缴费记录查询 | ✅ 历史记录（会员费+会议费两个标签） | — | P0 |
| M1-10 | 会员账号管理 | — | ✅ 查询/禁用/启用会员账号 | P0 |
| M1-11 | 手动开通会员 | — | ✅ 管理员为特定用户手动开通会员资格（绕过缴费流程） | P1 |

### 2.2 会员费缴纳业务流程

```
申请加入分会
    ↓
选择分会
    ↓
线下银行转账（至学会对公账户）
    ↓
上传缴费凭证（JPG/PNG ≤5MB）→ OCR识别
    ↓
上传电子发票（JPG/PNG/PDF ≤10MB）→ OCR识别+金额比对
    ↓
财务人工审核（支持批量通过/驳回）
    ├── 审核通过 → 会员资格生效（有效期1年）→ 自动通知
    └── 审核驳回 → 填写驳回原因 → 用户重新提交
```

### 2.3 会员状态流转

```
注册成功
  → 选择分会，申请加入
  → 上传凭证+发票 → 审核中
      ├── 审核通过 → 有效会员（有效期1年）
      ├── 审核驳回 → 已驳回（可重新提交）
      └── 未申请 → 可申请加入
  → 到期 → 已过期（可续费）
```

### 2.4 关键规则

| 规则 | 说明 |
|------|------|
| 一个邮箱一个账号 | 全局唯一，不可重复注册，不可修改 |
| 发票比对 | 凭证金额与发票金额必须一致，否则暂停流程 |
| 有效期1年 | 从审核通过日起算 |
| 到期提醒 | 到期前30天、7天自动通知 |

## 3. 模块二：会议管理模块

> **核心模块**：管理学术会议的完整生命周期：发布 → 通知 → 缴费 → 参会提交 → 统计导出

### 3.1 功能清单

| 功能编号 | 功能名称 | 用户端 | 管理端 | 优先级 |
|---------|---------|--------|--------|--------|
| M2-01 | 会议发布 | — | ✅ 总会/分会独立发布会议（富文本） | P0 |
| M2-02 | 会议推送通知 | — | ✅ 发布后自动推送给该分会所有会员 | P0 |
| M2-03 | 会议列表查看 | ✅ 仅可查看已加入分会的会议 | — | P0 |
| M2-04 | 会议详情查看 | ✅ 有效会员可查看详情和下载附件 | — | P0 |
| M2-05 | 会议费缴纳 | ✅ 线下转账→上传凭证→上传发票 | ✅ 审核 | P0 |
| M2-06 | 参会信息填写 | ✅ 基础信息+住宿+分会场+报告类型+报告标题 | ✅ 后台可查看统计 | P0 |
| M2-07 | 学术摘要提交 | ✅ 上传/替换.doc/.docx（≤10MB） | ✅ 批量导出 | P0 |
| M2-08 | 会议状态管理 | ✅ 自动流转（报名中→已截止→进行中→已结束） | — | P0 |
| M2-09 | 参会数据导出 | — | ✅ 参会人员+摘要+住宿统计Excel | P0 |
| M2-10 | 分会场配置 | — | ✅ 动态添加/编辑分会场选项 | P1 |
| M2-11 | 摘要修改截止日期设置 | — | ✅ 独立设置截止日期，超时置灰 | P1 |

### 3.2 会议费缴纳业务流程

```
前置条件：已上传该分会会员费凭证（不要求审核通过）

选择会议 → 选择身份（学生/教师/嘉宾）→ 确认金额
    ↓
线下银行转账
    ↓
上传会议费缴费凭证（OCR识别）→ 初审通过 → 可填写参会信息
    ↓
上传电子发票（7个工作日内，可申请延期）→ OCR识别+金额比对
    ↓
财务人工审核
    ├── 审核通过 → 参会资格生效
    └── 审核驳回 → 重新提交
```

### 3.3 参会信息提交流程

```
前置条件：会议费初审通过（已上传凭证）

填写表单：
├── 基础信息（姓名/性别/单位/身份，自动读取注册信息）
├── 住宿选择（单间/双人间/自行安排）
├── 分会场选择（后台配置的选项）
├── 报告类型（口头报告/展板报告/仅参会）
└── 报告标题（选择口头或展板时必填）

上传学术摘要（.doc/.docx ≤10MB，仅保留最新版本）

→ 提交成功 → 可无限次修改至截止日期
```

### 3.4 关键规则

| 规则 | 说明 |
|------|------|
| 前置条件 | 会员/非会员价格不一样 |
| 可见范围 | 仅可看到已加入分会的会议 |
| 附件下载 | 缴费后可下载通知PDF和摘要模板 |
| 单次缴纳 | 一次只能缴纳一个会议的注册费 |
| 发票时限 | 凭证上传后7个工作日内完成，超时可申请延期 |
| 摘要版本 | 只保留最后一次上传版本 |
| 修改截止 | 后台设置截止日期，超时按钮置灰 |

## 9. 风险提示与优化建议

### 9.1 技术风险

| 风险 | 影响 | 应对建议 |
|------|------|---------|
| **OCR智能识别准确率** | 凭证/发票图片质量参差不齐，识别可能出错，影响审核效率 | 接入百度OCR/腾讯OCR等成熟服务，人工审核做兜底 |
| **11个分会数据隔离** | 11套数据完全独立，权限逻辑复杂，容易出现权限漏洞 | 数据库层面用分会ID严格隔离，加强测试 |
| **文件存储容量增长** | 凭证/发票/摘要长期累积，存储成本持续增长 | 设置归档策略，超N年文件迁移至低成本存储 |
| **邮件送达率** | 自建邮件服务可能被判定为垃圾邮件 | 接入企业邮箱或SendCloud等专业服务 |
| **付款方与注册名不匹配** | 代缴/单位汇款/配偶转账等场景，OCR识别的付款方与用户实名不符 | 审核列表增加"付款方名称"OCR列，允许用户备注说明 |

### 9.2 用户体验风险

| 风险 | 说明 |
|------|------|
| 两段式审核流程复杂 | 凭证→发票→审核，流程较长，用户容易困惑 |
| 财务审核工作量增大 | 所有缴费需人工审核，规模扩大后压力较大 |
