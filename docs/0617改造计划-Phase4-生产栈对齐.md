## 6. 对齐方法论

```
1. 以 React 原型行为为「验收基准」
2. 以 agentic/03-api-docs 为「契约基准」
3. 后端先行 → 管理端 → 用户端
4. 每接口：请求示例 + 权限矩阵 + 错误码
5. 子模块内独立 commit，主仓库更新指针
```

---

## 7. 开发步骤

```
Step 1  读取 agentic/04-code-changes 2026-06-18~19 全部记录，列 diff
Step 2  后端：补 0617 与现有实现差距（若有）
Step 3  管理端 Vue：Statistics/Export/Audit 接 API
Step 4  用户端 Vue：会议/会员/参会资料接 API
Step 5  三端联调：总学会会议 → 四类缴费 → 统计 → 导出
Step 6  mvn compile + npm build（三端）
Step 7  更新 agentic 文档与子模块指针
```
