中国古生物学会会员管理系统 – 更新需求（基于最新沟通）

一、总体业务逻辑（变更点重点说明）

1.统一会员体系

只设一种会员身份：中国古生物学会会员。

会员费统一收取（一次缴费，成为学会会员）。

不再按分会单独收取会员费。

2.分会绑定机制

会员可自由选择绑定 0~11 个分会（例如 11 个专委会/分会）。

绑定后，自动接收该分会发布的会议通知、学术文章等。

可随时取消绑定，取消后不再接收该分会推送。

未绑定的分会，用户看不到其专属内容。

3.会议费独立收取

每个分会独立发布学术会议（每个会议周期约两年，时间表各异）。

会员按会议缴纳注册费，一场一场缴纳。

参加几个会议，就缴几次会议费。

财务手工对账，因此保持单笔单次缴费原则，不支持合并支付。

4.会员类型预留扩展

在会员模块中平行设计 3 个预留空位，用于后续增加：外籍会员、单位会员、荣誉会员。

这些会员类型可能享有不同权限或显示标签，当前阶段可仅做字段预留。

 

二、会员账号模块（不变）

仅支持邮箱注册，一个邮箱一个账号。

登录用邮箱+密码。

邮箱不可修改，其他信息可修改，需实名。

后台可禁用账号。

注册页字段：邮箱、密码、确认密码、姓名、性别、单位、职务（学生/老师/其他）、职称。

个人中心可修改姓名、性别、单位、职务、职称、密码。

 

三、会员费缴纳（统一收取）

用户注册登陆后可以选择加入会员，需缴纳中国古生物学会会员费（统一金额，如每年XX元，或可设置不同会员类型不同费率，目前先统一）。

缴费方式：线下银行转账 → 上传凭证（智能识别） + 上传电子发票（智能识别核对） → 财务人工审核。

审核通过后，会员资格生效，有效期自审核通过日起一年（或按年续费）。

会员到期自动失效，需续费才能继续享受所有服务（包括绑定分会、接收会议通知）。

缴费记录可在“我的缴费”中查看。

注意：不再有“分会会费”这个概念。用户只需缴纳一次学会会员费。

 

四、分会绑定与管理（新核心模块）

1. 分会列表展示

系统后台维护分会列表（现11个，可扩展）。

每个分会包含：名称、介绍、logo（可选）、专属会议列表、文章/通知。

2. 用户端“我的分会”页面

显示所有启用的分会。

每个分会旁有开关或按钮：【绑定】/【取消绑定】。

已绑定的分会：显示“已绑定”，可随时取消。

未绑定的分会：显示“绑定”，点击后立即绑定（无需审核，因为会员资格已有效）。

绑定条件：用户必须是有效会员（会员费已缴且在有效期内）。

非会员或过期会员无法绑定任何分会，需先续费。

3. 绑定后的效果

首页或“会议中心”只显示已绑定分会发布的会议和文章。

系统推送（站内通知+邮件）仅针对已绑定分会的新会议、重要通知等。

未绑定分会的会议、文章完全不可见。

 

4. 分会内容权限

下载会议通知、摘要模板等附件，除会员资格外，还需已绑定该分会。

后台可配置某些内容是否仅限绑定会员查看。

 

五、学术会议管理（按分会发布，独立缴费）

1. 会议发布

各分会独立发布学术会议（每个会议有独立页面）。

会议属性：

所属分会（必选）

会议名称、时间、地点、注册费金额（可分学生/教师/嘉宾等类别）

缴费截止日期

摘要提交截止日期

附件：会议通知PDF、摘要模板、野外路线旅游公司链接（不涉及收费）

 

2. 会议可见性

只有已绑定该分会的有效会员才能看到该会议。

在用户的“会议列表”中，只显示已绑定分会发布的最新会议（按时间倒序）。

未绑定分会的会议不会出现。

 

3. 会议费缴纳（单笔单次）

用户选择一个会议，点击【缴纳会议注册费】。

前提：

用户是有效会员（会员费未过期）

用户已绑定该分会

当前时间未超过缴费截止日期

进入缴费页：显示会议名称、注册费金额、收款账户信息。

上传缴费凭证（jpg/png ≤5MB） + 电子发票（jpg/png/pdf ≤10MB） → 提交。

一次只能提交一个会议的缴费申请。

财务人工审核（支持批量审核）。

审核通过后，该会议注册状态变为“已缴费”，获得参会资格。

 

4. 多个会议的处理

用户可以同时参加多个分会、多个会议，但每个会议独立缴费、独立审核。

缴费记录页面分会议展示，状态互不影响。

系统支持12个会议依次发布，因为各会议周期不同，时间上不会完全重叠，但技术上支持并行发布（用户可见多个待缴费会议）。

 

六、参会信息填写（与会议费审核通过挂钩）

前置条件：该会议的注册费已审核通过。

填写内容：

基础信息（姓名、性别、单位、身份）

住宿统计（单间/双人间/自行安排）

学术分会场（如会议设多个专场，后台配置）

报告类型（口头/展板/仅参会）及报告标题

学术摘要（.doc/.docx，有截止日期）

支持无限次修改，只保留最新版本。

每次修改自动邮件通知用户。

后台导出最新信息。

 

七、会员类型预留扩展

在会员表（或会员信息字段）中平行增加三个类型字段（或角色标识），用于将来区分：

预留位 用途示例 可能差异

类型1 外籍会员 可能单独设置会费金额、显示护照信息等

类型2 单位会员 以单位名义注册，可绑定多名联系人

类型3 荣誉会员 免会费、特殊标识

当前阶段可仅做数据模型预留，后台管理界面留出编辑入口，不实现业务逻辑差异。后续迭代再细化。

 

八、其他模块（基本沿用之前需求）

文件管理系统：凭证、发票、会议通知、摘要等上传下载，权限控制（会员+绑定分会）。

消息通知：站内通知+邮件，包括会议发布、缴费审核结果、缴费截止提醒、会员到期提醒等。

后台管理系统：

分会管理（增删改查）

会议管理（发布/编辑/下架）

会员管理（列表、续费、禁用、手动开通）

缴费审核（分会会费已无，只保留会议费审核）

数据导出（参会人员、缴费记录等）

统计展示：会员总数、各分会绑定人数、各会议缴费人数及金额（按职称分类）。

 

 

 

九、关键规则总结（与客户确认）

项目 规则

会员费 统一收取一次，成为学会会员

分会 会员自由绑定/解绑，最多11个，绑定后接收内容

会议费 按每个会议单独收取，参加几个缴几次

缴费方式 线下转账 + 上传凭证/发票 + 人工审核（不支持合并支付）

财务审核 单笔单次会议，方便人工对账

会议发布 各分会独立发布，时间表不同，系统支持并行展示

会员扩展 预留3个类型（外籍/单位/荣誉）

权限控制 查看会议、下载附件需：会员有效 + 已绑定该分会
