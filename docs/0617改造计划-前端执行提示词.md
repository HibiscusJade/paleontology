## 使用方式

1. 按阶段顺序执行，**确认前置条件**后再发下一条提示词。  
2. 每次只做一个阶段，避免 AI 跨阶段大范围改动。  
3. 阶段结束验证：`pnpm check`（必做），可选 `pnpm build`。  
4. 设计稿参考（若有）：`agentic/02-design/references/` 下 V3/V4 HTML。

### 前置条件

- Phase F0 完成  
- `pnpm dev` 可启动 website（:3000）

```

工作目录：paleontology-website-latest/

需求（MRD）：
- 总学会会议置顶可见、可报名（isSocietyAccessible，P0 已有则只补 UI）

- 两阶段缴费：凭证 → 发票 → confirmed 后才开放下载与参会表单
- 参会表单：报告类型、摘要 Word、住宿五选项、野外路线多选
- 摘要/住宿/野外 截止时间到达后 disabled + 提示文案

主要文件：
- client/src/pages/Services.tsx
- client/src/contexts/MembershipContext.tsx
- client/src/pages/Branches.tsx
- client/src/pages/SocietyAnnouncements.tsx（移除两档旧费用展示）
- shared/constants.ts

UI 要求：
- 对齐现有「地层次韵」风格（#002B49、#C41E3A、shadcn/ui）
- 状态文案参考 MembershipContext 中 CONFERENCE_STATUS / MEMBERSHIP_STATUS

禁止：
- 改 admin 项目
- 引入新 npm 依赖（除非必要且说明原因）
- 硬编码分会 ID/费用（从 @shared/constants 导入）

完成标准：
- pnpm check 通过
- 总学会会议未绑定 zgswxh 也可报名
- confirmed 前无盖章 PDF/摘要模板下载；confirmed 后有（admin 已上传 URL 时）

完成后：改动文件列表 + 手动验收步骤（3～5 条）。

```
