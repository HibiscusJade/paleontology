## 1. 项目总览

### 1.1 技术栈

| 技术 | 版本/方案 |
|------|----------|
| 前端框架 | React 19 |
| 构建工具 | Vite 7 |
| 类型系统 | TypeScript 5.6 |
| CSS框架 | Tailwind CSS 4 |
| UI组件库 | shadcn/ui (New York style) |
| 路由 | wouter (支持 hash/pushState 双模式) |
| 状态管理 | React Context (`MembershipContext`) |
| 图标 | Material Symbols Outlined + lucide-react |
| 表单 | react-hook-form + zod |
| 动画 | framer-motion |
| 图表 | recharts |

### 1.2 项目定位

当前原型是一个 **纯前端 SPA**，无后端 API。所有数据通过 `localStorage` 模拟持久化，所有交互（注册、缴费、审核、绑定）均在前端模拟完成。目标是**验证交互流程**，为后续后端开发提供参考。

### 1.3 核心模块

| 模块 | 文件 | 说明 |
|------|------|------|
| 共享常量 | `shared/constants.ts` | 会员费配置、状态枚举、标签映射、分会ID映射、会议-分会映射、宽限期常量 |
| 两阶段状态机 | `MembershipContext.tsx` | 8种会议状态 + 9种会员状态，含初审/终审分离 |
| 宽限期系统 | `MembershipContext.tsx` | 工作日截止日计算、自动逾期检测、截止日延期 |
| 会员到期处理 | `MembershipContext.tsx` | 分级处理（保留/冻结/删除）、续费恢复逻辑 |

### 2.2 两种布局模式

由 `PartyLayout.tsx` 统一控制：

| 模式 | 判断条件 | 应用于 | 特征 |
|------|---------|--------|------|
| **全宽** | `isFullWidthPage = true` | 首页、学会服务、学会简介、组织机构、学会沿革、历史相册、会员公告、国际交流、资料下载、规章条例、专业分会、个人中心 | 导航 + Hero + 面包屑 + 全宽内容 + 页脚 |

## 4. 会员体系（核心）

> **这是整个系统最核心的业务逻辑，所有交互都围绕会员状态展开。**

### 4.1 全局用户状态模型

```
用户状态层级（从底层到顶层）:

  ┌─────────────────────────────────────────────────────────┐
  │ 第3层：会议参与状态                                       │
  │   conferenceRegs[confId].status                          │
  │   unpaid → voucher_submitted → invoice_pending           │
  │         → invoice_overdue → invoice_submitted            │
  │         → confirmed                                      │
  │   (可驳回: voucher_rejected, invoice_rejected)           │
  ├─────────────────────────────────────────────────────────┤
  │ 第2层：分会绑定状态                                       │
  │   boundBranches: string[]                                │
  │   条件：已选择参与方式（非会员或正式会员）                   │
  │   到期时自动清空                                          │
  ├─────────────────────────────────────────────────────────┤
  │ 第1层：学会会员状态                                       │
  │   societyMembership.status                               │
  │   not_member → voucher_submitted → invoice_pending       │
  │             → invoice_overdue → invoice_submitted        │
  │             → active → expired                           │
  │   (可驳回: voucher_rejected, invoice_rejected)           │
  ├─────────────────────────────────────────────────────────┤
  │ 第0层：账号状态                                           │
  │   isLoggedIn: boolean                                    │
  │   currentUser: User | null                               │
  │   userType: "regular" | "non_member" | "member"         │
  └─────────────────────────────────────────────────────────┘
```

**层级依赖关系：**
- 第3层依赖第2层 + 第1层（参加会议需：已选择参与方式 + 已绑定该分会）
- 第2层依赖用户类型选择（绑定分会需：非会员或正式会员）
- 第1层依赖第0层（申请入会需：已登录）

### 4.2 账号注册

#### 4.2.1 注册入口（3个触发路径）

```
路径A：顶部导航栏 "登录" 按钮
  → 弹窗打开，初始 activeTab = "login"
  → 用户手动切换到 "注册" 标签

路径B：顶部导航栏 "加入会员" 按钮（未登录状态）
  → 弹窗打开，初始 activeTab = "register"

路径C：学会服务页 → 会员服务Tab → 未登录引导区
  → "登录账号" / "注册账号" 两个按钮
```

#### 4.2.2 注册表单字段与校验

| 字段 | 规则 | 错误提示 |
|------|------|---------|
| 所有带*字段 | 非空 | "请填写所有必填字段" |
| 密码 vs 确认密码 | 必须一致 | "两次输入的密码不一致" |
| 密码长度 | ≥ 6位 | "密码长度不能少于6位" |
| 邮箱 | 全局唯一（查 `MOCK_USER_DB`） | "该邮箱已被注册" |

**注册成功后：**
1. 用户数据写入 `localStorage` (`paleo_user_db`)
2. 自动登录（设置 `currentUser`）
3. 关闭弹窗
4. toast "注册成功！欢迎加入中国古生物学会。"
5. 添加通知

#### 4.2.3 注册后初始状态

```
注册完成后的默认状态:
  isLoggedIn          = true
  userType            = "regular"（普通用户，尚未选择路径）
  societyMembership   = { status: "not_member", history: [] }
  boundBranches       = []
  conferenceRegs      = {}
```

### 4.3 登录与登出

#### 4.3.1 登录流程

```
校验:
  ├── 邮箱未注册 → toast "该邮箱尚未注册"
  ├── 密码错误 → toast "密码错误，请重试"
  └── 正确 → 读取 localStorage 中的用户数据
           → 设置 currentUser
           → 恢复所有持久化状态
           → 首次登录时弹出双路径选择对话框
           → toast "登录成功"
```

#### 4.3.2 安全退出

```
触发: 用户下拉菜单 → "安全退出"（两处入口：导航栏 + 个人中心）

执行:
  1. 清除 currentUser → null
  2. isLoggedIn → false
  3. 清除 userType 和 membershipChoiceMade
  4. toast "已安全退出"
  5. 导航回首页
  6. localStorage 数据保留（下次登录恢复）
```

#### 4.3.3 账号注销

```
触发: 个人中心 → "注销账号"

执行:
  1. 两步确认（点击注销 → 红色警告确认区 → 确认注销）
  2. 永久删除账号及全部关联数据：
     - 从 paleo_user_db 删除
     - 清除所有 paleo_*_{email} 键
     - 清除登录状态
  3. 跳转至首页
  4. 不可撤销
```

#### 4.3.4 忘记密码

```
输入邮箱 → 点击"发送重置邮件" → toast 提示已发送
注意: 当前为模拟实现，不实际发送邮件
```

### 4.4 统一会费缴纳（两阶段审核）

#### 4.4.1 双路径选择（首次登录）

用户首次登录后，弹出 `MembershipChoiceDialog`：

- **路径 A（非会员）：** "作为非会员继续"→ `userType = "non_member"`，无需缴费，直接使用系统。可绑定分会、报名会议（非会员价 = 会员价×1.1）。后续可升级。
- **路径 B（正式会员）：** "成为正式会员"→ 进入两阶段缴费流程。

#### 4.4.2 完整状态机

```
                    ┌─────────────┐
        自动注册后   │ not_member  │
      ┌───────────→ │             │
      │             └──────┬──────┘
      │                    │ submitMembershipVoucher()
      │                    │ (上传凭证，提交初审)
      │                    ▼
      │             ┌─────────────────┐
      │     ┌───────│ voucher_submitted│  ← 凭证已上传，待初审
      │     │       └────────┬────────┘
      │     │                │
      │     │    ┌───────────┼───────────┐
      │     │    │                       │
      │     │    │ 初审通过               │ 初审驳回
      │     │    │ simApproveSociety     │ simRejectSociety
      │     │    │ Voucher()             │ Voucher(reason)
      │     │    ▼                       ▼
      │     │ ┌────────────────┐  ┌──────────────────┐
      │     │ │ invoice_pending │  │ voucher_rejected │
      │     │ │ (设置发票截止日)│  │ (重新上传凭证)    │
      │     │ └───────┬────────┘  └────────┬─────────┘
      │     │         │                    │
      │     │    ┌────┼────────┐           │
      │     │    │             │           │
      │     │    │ 正常上传     │ 逾期       │
      │     │    ▼             ▼           │
      │     │ ┌──────────┐ ┌──────────┐   │
```

#### 4.4.3 缴费流程详细交互（4步）

**进入条件：** 用户在会员服务Tab点击"立即申请入会"

**STEP 3: 上传凭证（阶段一）**

- OCR金额比对 → 一致后 → status = `invoice_submitted`

**截止日倒计时实现：**

```typescript
// 内联IIFE，在 invoice_pending 状态时渲染
{(() => {
  const deadline = reg.invoiceExtendedDeadline || reg.invoiceDeadline;
  if (!deadline) return null;
  const daysLeft = Math.ceil(
    (new Date(deadline).getTime() - new Date().getTime()) / (1000 * 60 * 60 * 24)
  );
  if (daysLeft <= 0) return <span className="text-red-600">已逾期</span>;
  if (daysLeft <= 3) return <span className="text-orange-600">还有{daysLeft}天</span>;
  return <span className="text-blue-600">还有{daysLeft}天</span>;
})()}
```
