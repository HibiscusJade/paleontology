## 1. 项目概述

### 1.1 项目背景

中国古生物学会及其下属11个专业分会需建设统一的新网站及会员管理系统，以支撑学会日常运营，重点解决以下问题：

- 会员注册、会费缴纳、会员资格管理的线上化
- 学术会议从发布到参会报名的全流程管理
- 党建文化内容的规范化展示与维护
- 满足审计整改要求，实现财务对账清晰可追溯

### 1.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **单账号全局通用** | 一个邮箱对应一个账号，全站所有分会通用 |
| **统一会员体系** | 只设一种会员身份——中国古生物学会会员，统一收取会员费 |
| **分会自由绑定** | 会员自由绑定/解绑多个分会（0~11个），绑定后接收该分会内容推送 |
| **权限隔离** | 仅已绑定该分会的有效会员可查看该分会专属会议和内容 |
| **单次缴费** | 会员费统一缴纳一次；会议费按每个会议单笔单次缴纳（符合财务审核要求） |
| **智能+人工审核** | 凭证和发票上传后，先智能识别，再人工审核 |
| **最新版本原则** | 参会信息和摘要文件只保留最后一次更新版本 |
| **自动通知** | 所有关键操作自动触发站内通知+邮件通知 |

### 6.2 会议服务

#### 6.2.1 会议发布

- 各分会独立发布学术会议（每个会议周期约两年，时间表各异）
- 系统支持并行展示多个会议（各分会时间表不同，12个会议可依次或并行发布）

#### 6.2.2 会议可见性

```
会议可见性规则：
  前提条件：用户是有效会员（会员费已缴且在有效期内）
         +
  用户已绑定该分会
         ↓
  方可看到该分会发布的会议
```

- 未绑定分会的会议**完全不可见**（不出现于任何列表）
- 绑定后立即可见该分会所有会议
- 取消绑定后，该分会会议从列表中消失

#### 6.2.3 会议属性

每个会议包含：

| 内容 | 说明 |
|------|------|
| 所属分会 | 必选，一个会议只属于一个分会 |
| 会议名称 | 会议标题 |
| 会议时间 | 起止日期 |
| 会议地点 | 举办地点 |

## 14. 统一会员体系·分会绑定·会议费——业务流程详解

### 14.1 账号体系

#### 注册规则

```
邮箱注册（全局唯一）
  → 填写实名信息
  → 注册成功
  → 全站通用（一个账号访问所有分会功能）
```

**约束：**
- 一个邮箱 = 一个账号，不可重复注册
- 邮箱不可修改
- 账号可被后台禁用，禁用后无法登录任何功能

---

### 14.2 统一会员费缴纳流程

#### 核心逻辑

```
用户注册登录
  → 选择【成为会员】
  → 缴纳中国古生物学会统一会员费（一次缴费）
  → 审核通过 → 成为有效会员（有效期1年）
  → 有效会员可绑定任意分会
```\n\n#### 详细步骤

**第1步：进入会员申请**

- 用户在个人中心或会员服务页点击【成为会员】
- 系统展示会员费金额（统一标准，如每年XX元）、收款账户信息
- 缴费方式：**线下银行转账**

**第2步：线下银行转账**

用户根据页面显示的收款账户信息，通过银行柜台或网银完成转账。

**第3步：上传缴费凭证**

| 要求 | 说明 |
|------|------|
| 格式 | JPG / PNG |
| 大小 | ≤ 5MB |
| 内容 | 银行转账回执单/截图 |

- 系统对凭证进行**智能识别**（OCR提取金额、收款方、付款方、日期）

**第4步：上传电子发票**

| 要求 | 说明 |
|------|------|
| 格式 | JPG / PNG / PDF |
| 大小 | ≤ 10MB |

- 系统对发票进行**智能识别**（OCR提取发票号码、金额、开票方、日期）
- 系统自动**比对**凭证金额与发票金额是否一致
- 比对通过 → 进入人工审核队列
- 比对不通过 → 提示用户重新上传

**第5步：财务人工审核**

- 管理员在后台看到待审核列表
- 支持**多选/全选+批量审核**
- 审核通过 → 会员资格生效，有效期从审核通过日起1年
- 审核驳回 → 填写驳回原因，通知用户重新提交

**关键限制规则：**

| 规则 | 说明 |
|------|------|
| 统一缴费 | 只缴纳一次，成为学会会员 |
| 不可合并 | 会员费和会议费不能合并支付 |
| 手动审核 | 所有缴费须经财务人工确认 |
| 单笔单次 | 保持单笔单次缴费原则，方便财务对账 |

### 14.3 分会绑定与管理

#### 核心逻辑

```
成为有效会员（会员费已缴且在有效期内）
  → 进入"我的分会"页面
  → 浏览11个分会
  → 自由【绑定】/【取消绑定】任意分会
  → 绑定后立即生效
  → 绑定后接收该分会的会议通知、学术文章
```

#### 用户端"我的分会"页面

**页面布局：**

- 顶部状态提示：
  - 有效会员 → 显示"您已是中国古生物学会有效会员，可自由绑定分会"
  - 非会员/过期 → 显示"请先缴纳会员费，方可绑定分会" + 【去缴费】按钮
- 分会卡片网格（11个分会）

**每个分会卡片显示：**

| 元素 | 说明 |
|------|------|
| 分会Logo | 可选，后台配置 |
| 分会名称 | 分会全称 |
| 分会简介 | 简短介绍 |
| 绑定状态 | 已绑定：绿色"已绑定" + 【取消绑定】按钮 |
|  | 未绑定：灰色"未绑定" + 【绑定】按钮 |
| 该分会会议数 | 当前进行中的会议数量 |

#### 绑定/解绑规则

| 操作 | 条件 | 效果 |
|------|------|------|
| **绑定** | 用户是有效会员（会员费已缴且在有效期内） | 立即生效，无需审核；可看到该分会会议和通知 |
| **取消绑定** | 无特殊条件 | 立即生效；该分会内容从列表中消失 |
| **自动解绑** | 会员到期自动失效 | 所有分会绑定自动失效；续费后需重新绑定 |

#### 绑定后的效果

1. **首页/会议中心：** 只显示已绑定分会发布的会议和文章
2. **系统推送：** 站内通知+邮件仅推送已绑定分会的新会议、重要通知
3. **内容隔离：** 未绑定分会的会议、文章完全不可见
4. **附件下载：** 需会员有效 + 已绑定该分会
5. **绑定数量：** 可绑定0~11个分会，无强制下限

#### 绑定状态流转

```
有效会员登录
  → 浏览分会列表
  → 点击【绑定】→ 立即绑定成功
  → 该分会内容可见
  → 点击【取消绑定】→ 确认 → 立即解绑
  → 该分会内容隐藏
  → 可随时重新绑定
```

### 14.4 会议费缴纳流程

#### 前置条件

- 用户是**有效会员**（会员费未过期）
- 用户**已绑定该分会**
- 当前时间**未超过缴费截止日期**\n\n#### 流程设计

```
┌─────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 选择会议 │ → │ 选择身份  │ → │ 确认金额  │ → │ 线下转账  │ → │ 上传凭证  │ → │ 上传发票  │ → │ 等待审核  │
└─────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
```

#### 详细步骤

**第1步：选择会议**

- 用户在"我的会议"或分会页面看到已绑定分会发布的会议
- 只有已绑定该分会的有效会员才能看到会议
- 点击进入会议详情页

**第2步：选择身份与金额**

会议注册费分三类：

| 身份 | 说明 |
|------|------|
| 学生 | 学生注册费标准 |
| 教师 | 教师注册费标准 |
| 嘉宾 | 嘉宾注册费标准 |

- 身份选择应与注册信息一致

**第3步：线下银行转账**

**第4步：上传缴费凭证**

- 格式：JPG / PNG，≤ 5MB
- 智能识别后，状态变更为"初审通过：可填写参会信息"

**第5步：上传电子发票**

- 格式：JPG / PNG / PDF，≤ 10MB
- 智能识别+比对通过 → 进入人工审核

**第6步：财务审核**

- 审核通过 → 获得参会资格
- 审核驳回 → 填写原因，用户重新提交

**关键限制规则：**

| 规则 | 说明 |
|------|------|
| 前置条件 | 有效会员 + 已绑定该分会 |
| 单次缴费 | 一次只能缴纳一个会议的注册费 |
| 截止日期 | 超时不可提交，按钮置灰 |
| 独立审核 | 每个会议独立审核，互不影响 |
| 合并禁止 | 不支持多个会议合并支付 |
| 多会议并行 | 用户可同时参加多个分会、多个会议，各自独立缴费 |

### 14.6 状态流转总图

#### 会员完整生命周期

```
用户注册
  → 申请成为会员
  → 线下转账
  → 上传缴费凭证（OCR识别）
  → 上传电子发票（OCR识别+比对）
  → 财务审核
      ├── 通过 → 中国古生物学会有效会员（有效期1年）
      │         → 可自由绑定/解绑任意分会
      │         → 可参加已绑定分会发布的会议（按会议独立缴费）
      │         → 到期 → 自动失效 → 所有绑定解除
      └── 驳回 → 重新提交
```

#### 会议参会完整生命周期

```
分会发布会议 → 仅对已绑定该分会的有效会员可见
  → 有效会员查看会议详情
  → 选择身份，转账
  → 上传缴费凭证（OCR识别）→ 初审通过 → 可填写参会信息
  → 上传电子发票 → OCR识别+比对
  → 财务审核
      ├── 通过 → 获得参会资格
      └── 驳回 → 重新提交
  → 参会信息 + 摘要可修改至截止日期
  → 会议结束后归入历史
```

#### 分会绑定生命周期

```
成为有效会员
  → 浏览分会列表
  → 点击【绑定】→ 立即绑定成功（无需审核）
  → 该分会内容可见
  → 【取消绑定】→ 立即解绑
  → 可随时重新绑定（仍然需是有效会员）
  → 会员到期 → 所有绑定自动解除
```

#### 17.4.2 审核工作流

（会员费审核 + 会议费审核，不再有"分会会费审核"）

```
缴费申请提交 → 系统OCR预处理 → 进入审核队列
                                 ↓
                          管理员查看列表
                          （凭证+发票并排预览）
                                 ↓
                    ┌───── 多选记录 ─────┐
                    ↓                    ↓
              【批量通过】           【批量驳回】
                    ↓                    ↓
              状态更新+通知          填写驳回原因+通知
```

审核列表功能：
- 筛选：按分会/会议/状态/日期
- 排序：按提交时间、金额
- 预览：点击查看凭证和发票原图
- 选中：复选框多选/全选当前页
- 一键操作：批量通过/批量驳回
- 驳回弹窗：文本框填写原因（必填）

| 类型3 | 荣誉会员 | 字段预留 | 免会费、特殊标识 |

- **当前阶段：** 仅做数据模型预留，后台管理界面留出编辑入口
- **后续迭代：** 实现不同会员类型的业务逻辑差异

## 19. 待确认问题与风险提示

### 19.1 业务流程待确认

| # | 问题 | 影响范围 | 优先级 |
|---|------|----------|--------|
| Q1 | 会员到期后，已绑定的分会全部自动解除。已缴费但尚未举行的会议参会资格是否保留？ | 会员+会议联动 | **高** |
| Q2 | 学生身份如何验证？是否需要上传学生证？还是仅靠用户自行声明？ | 会议费差额收费 | **高** |
| Q3 | 摘要修改截止日期后，如果有特殊情况（如报告人变更），是否有后台强制覆盖的能力？ | 参会信息 | 低 |
| Q4 | 双人间住宿是否需要填写同住人信息？ | 参会信息 | 低 |
| Q5 | 统一会员费金额是否对全部会员统一？还是后续扩展会员类型时会区分（如外籍会员不同费率、荣誉会员免费）？ | 会员费配置 | 中 |
| Q6 | 分会绑定/解绑是否记录操作日志？是否需要用户确认操作？ | 用户体验 | 低 |

### 19.3 技术风险提示

| # | 风险 | 说明 | 建议 |
|---|------|------|------|
| R1 | **OCR智能识别** | 缴费凭证和电子发票的OCR识别准确率受图片质量影响大 | 建议接入成熟的第三方OCR服务（如百度OCR/腾讯OCR），并在人工审核环节做兜底 |
| R2 | **实时统计性能** | 多个分会、多年份、交叉维度的实时统计，在大数据量下可能影响页面响应 | 建议使用缓存+定时计算 |
| R3 | **文件存储容量** | 凭证、发票、摘要长期累积，存储成本增长 | 建议设置归档策略 |
| R4 | **11个分会的独立性** | 11套分会页面结构相同但数据完全独立，后台管理逻辑需严格隔离 | 数据库层面用分会ID做数据隔离 |
| R5 | **邮件发送可靠性** | 自建邮件服务可能被邮箱服务商标记为垃圾邮件 | 建议接入企业邮箱或第三方发送服务 |
| R6 | **线下转账付款方与注册姓名不匹配** | 用户线下银行转账时，付款方名称可能与注册实名不一致 | 审核列表增加"付款方名称"OCR结果列，便于财务比对 |

### 19.4 需求合理性建议

| # | 建议 | 原因 |
|---|------|------|
| S2 | 建议增加在线支付选项（微信/支付宝） | 线下转账+上传凭证流程繁琐，增加财务人工审核负担 |
| S3 | 建议会议费身份与注册时填写的职务信息字段联动 | 避免信息不一致 |
| S4 | 建议党建模块部分内容复用网站统一CMS功能 | 避免重复开发 |
| S5 | 建议采用"缴费时锁定资格"规则 | 会员到期后不追溯取消已报名会议资格 |
