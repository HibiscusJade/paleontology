### 1. 项目概述与核心原则

- **项目背景：** 建设中国古生物学会（含11个分会）统一网站及会员管理系统，解决会员服务、会议管理、党建宣传及财务审计问题。
- **核心设计原则：**
  - **单账号全局通用：** 一个邮箱账号，全站通用。
  - **权限隔离：** 分会间数据（会员、会议）严格隔离。
  - **单次缴费：** 会费、会议费均单笔单次缴纳，符合审计要求。
  - **智能+人工审核：** 凭证/发票先OCR识别，再人工审核。
  - **最新版本原则：** 参会信息/摘要只保留最终版本。
  - **全流程通知：** 关键操作自动触发站内信+邮件。

### 2. 模块一：会员服务

#### 2.1 账号与注册

- **用户端（前台）**
  - **注册：** 使用邮箱注册，填写姓名、性别、单位、身份（学生/老师/其他/职称）等信息。邮箱全局唯一且不可修改。密码需6位以上。
  - **登录：** 首页右上角提供登录入口。
  - **密码重置：** 通过注册邮箱找回密码。
- **管理端（后台）**
  - **账号管理：** 管理员（学会总管理员）可查询所有会员账号，并可执行“禁用/启用”操作。禁用后账号无法登录。
  - **操作日志：** 记录账号状态变更操作。

#### 2.2 分会加入与会费缴纳

- **用户端（前台）**
  - **申请加入：** 在“组织机构”或“会员服务”页面浏览分会列表，点击【申请加入】。
  - **缴费流程：**
    1. **确认信息：** 确认分会名称、会费金额（统一标准）、收款账户信息。
    2. **线下转账：** 用户通过银行转账至学会对公账户。
    3. **上传凭证：** 上传转账回执（JPG/PNG，≤5MB）。系统OCR识别后，此步骤完成即可进入会议费注册。
    4. **上传发票：** 在凭证上传后，按要求上传电子发票（JPG/PNG/PDF，≤10MB）。系统OCR识别并比对金额。
    5. **等待审核：** 提交后状态变为“审核中”。
  - **状态查看：** 在“个人中心”查看申请状态（审核中/已驳回/有效会员/已过期）。
- **管理端（后台）**
  - **会费配置：** 学会总管理员可设置各分会的会费金额。
  - **缴费审核：**
    - **列表视图：** 按分会、状态、日期筛选待审核记录。同时展示用户上传的凭证和发票缩略图。
    - **智能比对：** 系统高亮显示OCR识别出的金额、双方账号，并与用户信息、应收金额做比对提示。
    - **审核操作：** 支持单条或批量审核（通过/驳回）。驳回需填写原因。
    - **手动开通：** 财务审核员或总管理员可为特定用户手动开通某分会会员资格（用于处理线下转账等特殊情况），绕过线上缴费流程。
  - **审核通过后：** 会员资格生效，有效期为审核通过日起1年。系统自动触发邮件和站内信通知。

#### 2.3 个人中心与会员状态

- **用户端（前台）**
  - **信息管理：** 可修改除邮箱外的个人信息，修改密码需验证原密码。
  - **我的分会：** 列表展示已加入/申请中的分会，每个分会卡片清晰显示会员状态（有效/审核中/已过期/已驳回）及有效期。
  - **我的会议：** 展示已报名/关注的会议及状态（报名中/已缴费/已结束）。
  - **缴费记录：** 分为“会费记录”和“会议费记录”两个标签页，展示历史缴费明细。
  - **消息中心：** 接收所有站内通知。
- **管理端（后台）**
  - **会员查询：** 管理员可按邮箱、姓名、分会、会员状态等维度查询会员。

### 3. 模块二：会议服务

#### 3.1 会议生命周期管理

- **用户端（前台）**
  - **会议列表：** 在学会首页、分会首页展示会议卡片。用户仅能看到其已加入分会的会议。
  - **会议详情：** 展示会议通知详情、时间、地点、注册费标准。**附件下载**（通知PDF、摘要模板）需用户是该分会“有效会员”。
  - **会议状态：** 页面直观展示会议状态（报名中/进行中/已截止/已结束）。
- **管理端（后台）**
  - **会议发布：** 分会管理员可创建会议，填写详细内容（富文本），上传通知PDF和摘要模板。
  - **参数配置：** 设置学生/教师/嘉宾三类注册费标准、缴费截止日期、摘要修改截止日期。
  - **分会场配置：** 可动态添加/编辑学术分会场选项（如“01专场：古脊椎动物学”）。
  - **权限推送：** 会议发布后，系统自动向该分会所有会员（无论是否缴费）发送通知。但详情页访问和附件下载受会员状态限制。

#### 3.2 会议注册费缴纳

- **用户端（前台）**
  - **前置条件：** 必须已上传该分会的**会员费缴费凭证**。
  - **缴费流程：**
    1. **选择身份：** 选择学生/教师/嘉宾，系统自动计算金额。（学生身份可能需要额外验证）
    2. **线下转账：** 向会议指定账户转账。
    3. **上传凭证：** 上传转账回执。审核通过后（初审通过），状态变更为“可填写参会信息”。
    4. **上传发票：** 凭证上传后7个工作日内，必须上传电子发票。超时可申请延期。
    5. **等待终审：** 财务审核凭证与发票。
- **管理端（后台）**
  - **审核流程：** 与会员费审核类似，支持OCR识别、比对、批量审核。审核通过后，该用户的参会资格正式生效。
  - **例外处理：** 若会员费审核最终被驳回，管理员需人工处理已缴纳的会议费（建议规则：用户可选择退款或保留至会员费审核通过，需确认 Q1）。

#### 3.3 参会信息与摘要提交

- **用户端（前台）**
  - **前置条件：** 会议费“初审通过”。
  - **提交表单：**
    - **基础信息：** 姓名、性别、单位自动读取，身份（教师/学生/嘉宾）与缴费时联动。
    - **住宿统计：** 选择单间/双人间/自行安排。
    - **学术参与：** 选择口头报告/展板报告/仅参会。若选择报告，需填写报告标题。
    - **摘要上传：** 上传 .doc/.docx 文件 (≤10MB)。可随时替换，系统仅保留最新版本。
  - **修改限制：** 在管理员设置的“摘要修改截止日期”前，可无限次修改参会信息和摘要。
- **管理端（后台）**
  - **截止日期设置：** 为每个会议独立设置“参会信息/摘要修改截止日期”，超时后前端提交按钮将置灰。
  - **数据查看与导出：**
    - **参会统计：** 实时统计总参会人数、报告人数、口头/展板报告数量。
    - **住宿统计：** 交叉统计（男单间×N，男双人间×N，女单间×N，女双人间×N）。
    - **摘要导出：** 可按分会场顺序，批量导出所有参会者的最新摘要文件。
  - **特殊操作：** 管理员可后台强制修改某用户的参会信息或替换其摘要（用于处理截止日期后的特殊情况）。

### 4. 模块三：内容管理（CMS）[里面所有模块都是配置]

> **说明：** 此模块负责所有展示型内容的维护。管理端的任何编辑，在前端对应页面实时生效。

#### 4.1 学会简介与静态页

- **前端（展示）**
  - **子栏目：** 学会概况、学会章程、现任/历任领导、理事会/监事会、秘书处、历史沿革、发展规划、获奖成果等。
  - **展示形式：** 富文本、图文列表、人员卡片、时间线等。
- **后端（管理）**
  - **富文本编辑器：** 支持文字排版、图片/附件上传、表格、视频嵌入、HTML源码编辑。
  - **人员管理：** 提供专门的人员列表管理界面，可增删改成员信息（姓名、照片、职务、简介、任期）。
  - **相册管理：** 支持批量上传、排序、删除图片。

#### 4.2 新闻/公告动态

- **前端（展示）**
  - **首页：** 展示“新闻动态”和“通知公告”列表（图文混排或标题列表）。
  - **列表页：** 分栏目展示所有动态，支持按时间、热度等排序。
  - **详情页：** 展示完整内容。
- **后端（管理）**
  - **发布管理：** 管理员可发布、编辑、置顶、下架文章/公告。
  - **分类管理：** 支持多级分类（如：学会新闻 -> 学术交流）。
  - **附件管理：** 可在文章中插入文件下载链接。

#### 4.3 党建文化专栏

- **前端（展示）**
  - **子栏目：** 通知公告、党群机构、党委纪委、党建工作、组织生活（三会一课等）、党员队伍建设、理论学习专栏、党建专题、先进典型、违法违纪举报、下载中心。
  - **展示形式：** 文章列表、结构化页面（如党员队伍建设下设多个子页）、外部链接跳转。
- **后端（管理）**
  - **复用CMS能力：** 通知公告、工作动态、先进典型等栏目复用“新闻/公告动态”的管理功能。
  - **专题管理：** 支持创建自定义专题（如“党纪学习教育”），可将相关文章、活动动态、图片归集到一个专题页面下。
  - **结构化内容管理：** 对“党群机构”、“党员队伍建设”等结构复杂的页面，提供独立的表单式编辑界面，方便维护人员、机构、制度等信息。
  - **文件管理：** “下载中心”提供独立文件库，支持分类和权限控制（如仅党员可下载）。

#### 4.4 科学传播与国际交流

- **前端（展示）**
  - **科学传播：** 下设期刊服务、科普基地、科普文章/视频、化石保护等子栏目。
  - **国际交流：** 下设交流动态、国际会议、合作机构等子栏目。
- **后端（管理）**
  - **多格式内容支持：** 文章（图文）、视频（支持嵌入代码）、链接（外站期刊）、列表（科普基地）。
  - **统一管理：** 所有子栏目的内容均通过CMS的“栏目管理”和“内容发布”功能进行维护。

### 5. 模块四：组织机构管理

#### 5.1 分会独立站点管理

- **前端（展示）**
  - **入口：** “组织机构”页左侧为11个分会列表，右侧为总会架构图。
  - **独立站点：** 每个分会拥有独立的子站点，包含分会首页、概况、理事会、工作动态、通知公告、历史沿革、科学传播、下载中心等完整栏目。
  - **数据隔离：** 用户在分会页面下的操作（如申请加入、下载附件）仅与该分会相关。
- **后端（管理）**
  - **分会管理：** 学会总管理员可新增/编辑/禁用分会。禁用后，该分会在前端入口隐藏，所有相关功能（会员、会议）停用。
  - **分会内容管理：** 每个分会自动拥有独立的CMS权限空间。分会管理员登录后，只能编辑自己分会的页面内容（所有子栏目），无法修改总会或其他分会的内容。
  - **分会人员管理：** 分会管理员可管理本分会的理事会成员列表。

### 6. 模块五：系统与支撑功能

#### 6.1 消息通知系统

- **前端（接收）**
  - **站内信：** 首页铃铛图标+红点，展示所有通知（如审核结果、新会议发布、缴费提醒）。
  - **邮件通知：** 所有关键操作（注册、审核、缴费成功、截止提醒）均发送邮件到注册邮箱。
- **后端（配置与发送）**
  - **模板管理：** 提供10种邮件模板（注册成功、审核通过/驳回、提醒等），支持编辑。
  - **发送日志：** 记录所有邮件的发送状态、时间、收件人，发送失败自动重试3次。
  - **发送服务：** 建议接入企业邮箱或专业邮件服务（SendCloud），确保送达率。

#### 6.2 文件管理与权限

- **前端（使用）**
  - **上传：** 在缴费、提交摘要等场景上传文件，前端有格式和大小校验。
  - **下载：** 点击附件链接，后端进行权限校验后才返回文件流。
- **后端（存储与鉴权）**
  - **存储规则：** 文件统一存储，命名规则为`类型_用户ID_时间戳.后缀`，路径存入数据库。
  - **权限校验核心逻辑：**
    - **分会附件（会议通知、摘要模板）：** 必须校验当前用户是否为该分会的**有效会员**（会员费已缴且在有效期内）。
    - **个人文件（自己的凭证、发票、摘要）：** 登录用户只能访问自己的文件。
    - **党建文件：** 登录用户即可访问（特殊文件可配置“党员”角色）。
  - **生命周期管理：**
    - **替换：** 用户上传新文件（如新摘要），系统自动删除旧文件。
    - **级联删除：** 后台删除会议或分会时，关联的所有文件需一并清理。
    - **归档策略：** 可配置策略，将超过N年的文件自动迁移至低成本存储。

#### 6.3 数据统计与导出

- **前端（查看与下载）**
  - **统计面板：** 学会/分会管理员可在后台首页看到关键数据图表（如当月缴费趋势、各分会会员数）。
  - **导出功能：** 在“会员列表”、“缴费记录”、“参会人员”等页面提供“导出Excel”按钮。
- **后端（计算与生成）**
  - **实时统计：** 会员费/会议费总额、人数等核心指标需实时或准实时（缓存30分钟）计算。
  - **复杂统计：** 住宿交叉统计（男单间等）、按分会场的摘要分类导出等功能需在后端实现聚合查询和文件打包。
  - **导出内容：**
    - **会员导出：** 完整会员信息列表。
    - **缴费记录导出：** 含凭证号、金额、状态、审核时间，供财务对账。
    - **参会人员导出：** 含所有参会信息字段，并生成摘要文件的下载链接列表。

#### 6.4 后台权限与安全

- **用户端（无感知）**
  - 所有前端操作都受后端权限和状态校验保护。
- **后端（管理）**
  - **角色定义：**
    - **学会总管理员 (5人)：** 拥有全站所有模块的读写权限（CMS+功能数据+系统配置）。
    - **分会管理员 (每分会2人)：** 拥有所属分会的CMS编辑权限和本分会会员/会议/缴费数据的管理权限。
    - **财务审核员 (若干)：** 仅拥有所有分会的缴费审核权限（只读其他数据，无导出权限）。
  - **操作日志：** 对所有“增、删、改、审、导”操作进行全记录，包括操作人、时间、IP、操作详情，满足审计要求。
