## 1. 项目概述

### 1.1 项目背景

中国古生物学会及其下属11个专业分会需建设统一的新网站及会员管理系统，以支撑学会日常运营，重点解决以下问题：

- 会员注册、会费缴纳、会员资格管理的线上化
- 学术会议从发布到参会报名的全流程管理
- 党建文化内容的规范化展示与维护
- 满足审计整改要求，实现财务对账清晰可追溯

### 1.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **单账号全局通用** | 一个邮箱对应一个账号，全站所有分会通用 |
| **双路径会员体系** | 用户首次登录后自主选择：作为"非会员"直接使用（会议费较高），或成为"正式会员"（需缴费验证，享会员价） |
| **统一会员身份** | 正式会员身份统一为"中国古生物学会会员"，统一收取会员费 |
| **分会自由绑定** | 非会员和正式会员均可自由绑定/解绑多个分会（0~11个），绑定后接收该分会内容推送 |
| **权限隔离** | 仅已绑定该分会的用户可查看该分会专属会议和内容 |
| **单次缴费** | 会员费统一缴纳一次；会议费按每个会议单笔单次缴纳（符合财务审核要求） |
| **两阶段审核** | 缴费审核分为凭证初审和发票终审两个阶段，确保财务合规 |
| **智能+人工审核** | 凭证和发票上传后，先智能识别（OCR），再人工审核 |
| **最新版本原则** | 参会信息和摘要文件只保留最后一次更新版本 |
| **自动通知** | 所有关键操作自动触发站内通知+邮件通知 |

### 1.3 整体业务逻辑

```
用户注册 → 首次登录 → 选择参与方式
  ├── 路径A：非会员 → 直接绑定分会 → 报名会议（非会员价）
  └── 路径B：正式会员 → 缴纳统一会员费（两阶段审核）
        → 成为有效会员（有效期1年）
        → 绑定分会 → 报名会议（会员价）

会员到期 → 自动失效 → 所有绑定失效 → 会议资格分级处理
```

### 1.4 模块总览

网站共设11个一级导航栏目，其中学会服务为交互式核心模块，党建文化为独立子系统的核心模块。

```

     → 学会沿革 / 历史相册 / 会员公告 / 国际交流 / 资料下载 / 规章条例

├── 会议服务（会议发布/会议费缴纳/参会提交）
├── 科学传播
└── 科技奖励

党建文化
```

## 6. 模块四：学会服务

### 6.1 会员服务

#### 6.1.1 会员注册

```
注册流程：
邮箱注册 → 填写个人信息 → 提交 → 注册成功 → 跳转登录
```

| 字段 | 说明 |
|------|------|
| 邮箱 | 必填，不可修改，全局唯一 |
| 密码 | 必填，6位以上 |
| 确认密码 | 必填，需一致 |
| 姓名 | 必填，实名制 |
| 性别 | 必填，单选 |
| 单位 | 必填 |
| 职务信息 | 选填：学生 / 老师 / 其他 / 职称 |

#### 6.1.2 个人中心

- 显示个人信息（邮箱不可编辑，其余可修改）
- 修改密码需输入原密码
- **安全退出与注销：** 导航栏用户下拉菜单与个人中心均提供"安全退出"。个人中心另提供"注销账号"——两步确认后永久删除账号及全部关联数据，不可撤销。
- **我的分会**
- 我的会议列表
- 缴费记录（会员费 + 会议费，两个标签页）
- 消息中心

#### 6.1.3 双路径会员体系与统一会员费缴纳

用户注册登录后，首次进入系统弹出选择对话框：
- **路径 A（非会员）：** 选择"作为非会员继续"→ 无需缴费，直接使用系统。可绑定分会、报名会议（按非会员价，会员价×1.1）。
- **路径 B（正式会员）：** 选择"成为正式会员"→ 缴纳中国古生物学会统一会员费，经两阶段审核后成为有效会员（有效期1年），享受会员价参会。

详见 [§14.2 双路径会员体系与统一会员费缴纳](#142-双路径会员体系与统一会员费缴纳两阶段审核)

#### 6.1.4 会员状态

| 状态码 | 页面显示 | 颜色 | 可操作 |
|--------|----------|------|--------|
| `not_member` | "尚未入会" | 灰色 | 申请入会 |
| `voucher_submitted` | "凭证初审中" | 黄色 | 等待 |
| `voucher_rejected` | "凭证被驳回" | 红色 + 原因 | 重新提交凭证 |
| `invoice_pending` | "待上传发票" | 蓝色 + 截止日倒计时 | 上传发票 |
| `invoice_overdue` | "发票逾期" | 橙色 | 补传发票 |
| `invoice_submitted` | "发票终审中" | 黄色 | 等待 |
| `invoice_rejected` | "发票被驳回" | 红色 + 原因 | 重新提交发票 |
| `active` | "会员资格有效" | 绿色 + 有效期 | 续费 |
| `expired` | "已过期" | 灰色 | 续费 |

### 6.2 会议服务

#### 6.2.1 会议发布

- 各分会独立发布学术会议
- 系统支持并行展示多个会议

#### 6.2.2 会议可见性

```
会议可见性规则：
  前提条件：用户已选择参与方式（非会员或正式会员）
         +
  用户已绑定该分会
         ↓
  方可看到该分会发布的会议
```

- 未绑定分会的会议**完全不可见**（不出现于任何列表）
- 绑定后立即可见该分会所有会议
- 取消绑定后，该分会会议从列表中消失

#### 6.2.3 会议属性

每个会议包含：

| 内容 | 说明 |
|------|------|
| 所属分会 | 必选，一个会议只属于一个分会 |
| 会议名称 | 会议标题 |
| 会议时间 | 起止日期 |
| 会议地点 | 举办地点 |
| 通知详情 | 富文本内容 |
| 通知PDF附件 | 仅已绑定该分会的用户可下载 |
| 摘要模板附件 | 仅已绑定该分会的用户可下载 |
| 注册费信息 | 会员价与非会员价两套标准 |
| 缴费截止日期 | 超时不可提交 |
| 摘要提交截止日期 | 超时不可修改摘要 |

#### 6.2.4 会议状态

| 状态 | 显示 | 触发条件 |
|------|------|----------|
| 报名中 | 正常展示，可缴费 | 会议发布后至截止日期前 |
| 已截止 | "缴费已截止" | 超过缴费截止日期 |
| 进行中 | 会议日期范围内 | — |
| 已结束 | "已结束"，归入历史会议 | 会议日期过后 |

#### 6.2.5 用户侧会议卡片

| 卡片状态 | 标签颜色 | 可操作 |
|----------|----------|--------|
| 未缴费 (`unpaid`) | 灰色 | 缴纳会议注册费 |
| 凭证初审中 (`voucher_submitted`) | 黄色 | 等待 |
| 凭证被驳回 (`voucher_rejected`) | 红色 | 重新上传凭证 |
| 待上传发票 (`invoice_pending`) | 蓝色 + 截止日倒计时 | 填写参会信息 / 上传发票 |
| 发票逾期 (`invoice_overdue`) | 橙色 | 补传发票 |
| 发票终审中 (`invoice_submitted`) | 黄色 | 等待 |
| 发票被驳回 (`invoice_rejected`) | 红色 | 重新上传发票 |
| 已确认 (`confirmed`) | 绿色 | 修改参会信息 |

#### 6.2.6 会议内容权限

| 内容 | 访问条件 |

## 14. 统一会员体系·分会绑定·会议费——业务流程详解

### 14.1 账号体系

#### 注册规则

```
邮箱注册（全局唯一）
  → 填写实名信息
  → 注册成功
  → 全站通用（一个账号访问所有分会功能）
```

**约束：**
- 一个邮箱 = 一个账号，不可重复注册
- 邮箱不可修改
- 账号可被后台禁用，禁用后无法登录

#### 安全退出与账号注销

- **安全退出：** 导航栏用户下拉菜单和个人中心均提供。清除登录会话与双路径选择状态，自动跳转首页。localStorage 数据保留。
- **账号注销：** 个人中心提供，两步确认后永久删除账号及全部关联数据（用户数据库、个人资料、会员资格、分会绑定、会议报名、通知消息），不可撤销，跳转首页。

### 14.2 双路径会员体系与统一会员费缴纳（两阶段审核）

#### 核心逻辑

```
用户注册登录 → 首次进入系统 → 弹出选择对话框
  ├── 路径A（非会员）：选择"作为非会员继续"
  │     → 无需缴费，直接使用系统
  │     → 可绑定分会、报名会议（按非会员价 = 会员价×1.1）
  │     → 后续可随时升级为正式会员
  │
  └── 路径B（正式会员）：选择"成为正式会员"
        → 缴纳中国古生物学会统一会员费
        → 【阶段一：凭证初审】
            上传转账凭证 → 财务初审 → 初审通过（可绑定分会）
        → 【阶段二：发票终审】
            上传电子发票（必传）→ OCR比对 → 财务终审 → 成为有效会员（有效期1年）
```

#### 会费配置

会员费金额**后台可配置**，支持按年度独立设置、按会员类型分别定价：

| 会员类型 | 默认年费 |
|----------|----------|
| 普通会员（standard） | ¥200 |
| 学生会员（student） | ¥100 |
| 单位会员（corporate） | ¥5000 |

运行时通过 `getMembershipFee(memberType)` 从配置读取，fallback 到默认值。

#### 会员费两阶段状态机

```
not_member
  → [转账 + 上传凭证] → voucher_submitted
    → [财务初审]
      ├── 通过 → invoice_pending（设置7工作日发票截止日）
      │   ├── [上传发票 + OCR比对] → invoice_submitted
      │   │   → [终审]
      │   │     ├── 通过 → active（正式会员，有效期1年）
      │   │     └── 驳回 → invoice_rejected → 重新上传发票
      │   └── [逾期] → invoice_overdue → 会员资格锁定（仍可上传）
      └── 驳回 → voucher_rejected → 重新上传凭证
```

#### 详细步骤

**第1步：进入会员申请**
- 用户在个人中心或会员服务页点击【成为会员】
- 系统展示会员费金额（从配置读取）、收款账户信息
- 缴费方式：**线下银行转账**

**第2步：线下银行转账**
用户根据页面显示的收款账户信息，通过银行柜台或网银完成转账。

**第3步：上传缴费凭证（阶段一：凭证初审）**

| 要求 | 说明 |
|------|------|
| 格式 | JPG / PNG |
| 大小 | ≤ 5MB |
| 内容 | 银行转账回执单/截图 |

- 系统对凭证进行智能识别（OCR提取金额、收款方、付款方、日期）
- 提交后状态变更为 `voucher_submitted`

**第4步：上传电子发票（阶段二：发票终审）**

| 要求 | 说明 |
|------|------|
| 格式 | JPG / PNG / PDF |
| 大小 | ≤ 10MB |
| 必传 | **是** |
| 截止时间 | 凭证初审通过日 + **7 个工作日**（宽限期） |

- 前置条件：凭证初审通过（状态为 `invoice_pending`）
- 系统展示发票上传截止日倒计时
- 系统对发票进行OCR识别并比对凭证金额
  - 比对通过 → 状态变更为 `invoice_submitted`，进入人工终审队列
  - 比对不通过 → 提示用户重新上传

**宽限期规则：**

| 规则 | 说明 |
|------|------|
| 截止日计算 | 凭证初审通过日 + 7个工作日（跳过周六日）|
| 逾期状态 | 超时未上传 → 状态自动变更为 `invoice_overdue`，会员资格锁定 |
| 逾期后处理 | 仍可上传发票；上传后正常进入终审流程 |
| 截止日提醒 | 距截止日 ≤ 3天：系统自动发送提醒通知 |
| 手动延期 | 财务后台可为单个缴费记录设置 `invoiceExtendedDeadline` |

**第5步：财务终审**
- 管理员在后台审核队列中查看发票（终审队列）
- 支持多选/全选+批量审核
- 终审通过 → 会员资格生效，有效期从审核通过日起1年
- 终审驳回 → 填写驳回原因，状态变更为 `invoice_rejected`

**关键限制规则：**

| 规则 | 说明 |
|------|------|
| 统一缴费 | 只缴纳一次，成为学会会员 |
| 不可合并 | 会员费和会议费不能合并支付 |
| 两阶段审核 | 凭证初审 → 发票终审，两阶段独立审核 |
| 发票必传 | 所有缴费须上传发票，宽限期7工作日 |
| 手动审核 | 所有缴费须经财务人工确认 |
| 单笔单次 | 保持单笔单次缴费原则，方便财务对账 |
| 金额可配置 | 会员费按年度+类型从后台配置读取 |

### 14.3 分会绑定与管理

#### 核心逻辑

```
已选择参与方式（非会员或正式会员）
  → 进入"我的分会"页面
  → 浏览11个分会
  → 自由【绑定】/【取消绑定】任意分会
  → 绑定后立即生效
  → 绑定后接收该分会的会议通知、学术文章
```

#### 用户端"我的分会"页面

- 顶部状态提示：非会员/有效会员显示引导绑定文字；普通用户显示"请先选择参与方式"
- 分会卡片网格（11个分会），每张显示：Logo、名称、简介、绑定状态开关

#### 绑定/解绑规则

| 操作 | 条件 | 效果 |
|------|------|------|
| **绑定** | 已选择参与方式（非会员或正式会员） | 立即生效，无需审核；可看到该分会会议和通知 |
| **取消绑定** | 无特殊条件 | 立即生效；该分会内容从列表中消失 |
| **自动解绑** | 会员到期自动失效 | 所有分会绑定自动失效；续费后需重新绑定 |

#### 绑定后的效果

1. **首页/会议中心：** 只显示已绑定分会发布的会议和文章
2. **系统推送：** 站内通知+邮件仅推送已绑定分会的内容
3. **内容隔离：** 未绑定分会的会议、文章完全不可见
4. **附件下载：** 需已绑定该分会
5. **绑定数量：** 可绑定0~11个分会，无强制下限

#### 会员到期后的处理

会员到期后所有分会绑定自动解除（`boundBranches = []`）。续费后**不自动恢复**绑定——用户需手动重新选择。

### 14.4 会议费缴纳流程（两阶段审核）

#### 前置条件

- 用户已选择参与方式（非会员或正式会员）
- 正式会员需会员资格有效（active / invoice_pending / invoice_submitted）
- 用户**已绑定该分会**
- 当前时间**未超过缴费截止日期**

#### 会议费双价

每场会议设置两个价格：会员价与非会员价。非会员价 = 会员价 × 1.1，向上取整。会议卡片仅显示与用户身份对应的价格；会议详情通知页同时展示双价。

非会员缴费时显示："升级为正式会员（¥200/年）后，本次会议可节省 ¥X"。

#### 会议费两阶段状态机

```
unpaid
  → [上传凭证] → voucher_submitted
    → [财务初审]
      ├── 通过 → invoice_pending（解锁参会信息填写 + 设置7工作日发票截止日）
      │   ├── [填写参会信息]
      │   ├── [上传发票 + OCR比对] → invoice_submitted
      │   │   → [终审]
      │   │     ├── 通过 → confirmed（报名完成）
      │   │     └── 驳回 → invoice_rejected → 重新上传发票
      │   └── [逾期] → invoice_overdue（参会信息锁定，仍可上传发票）
      └── 驳回 → voucher_rejected → 重新上传凭证
```

#### 详细步骤

**第1-3步：** 选择会议 → 选择身份与金额 → 线下银行转账。

**第4步：上传缴费凭证（阶段一）**
- 格式：JPG / PNG，≤ 5MB
- 提交后状态变更为 `voucher_submitted`

**初审通过：** 状态变更为 `invoice_pending`，解锁参会信息填写权限，设置发票上传截止日。

**第5步：上传电子发票（阶段二）**
- 格式：JPG / PNG / PDF，≤ 10MB，**必传**
- OCR识别发票金额 + 与凭证金额比对
- 宽限期：7 个工作日
- 逾期后：状态变更为 `invoice_overdue`，参会信息锁定，仍可上传

**第6步：财务终审**
- 终审通过 → 状态变更为 `confirmed`，获得参会资格
- 终审驳回 → 状态变更为 `invoice_rejected`

#### 状态→UI 对照表

| 状态 | 缴费按钮 | 参会信息 | 发票上传 | 状态标签颜色 |
|------|---------|---------|---------|------------|
| `unpaid` | 【立即缴纳注册费报名】| 隐藏 | 隐藏 | 灰色 |
| `voucher_submitted` | 隐藏 | 锁定 | 隐藏 | 黄色 |
| `voucher_rejected` | 【重新上传凭证】| 隐藏 | 隐藏 | 红色 |
| `invoice_pending` | 隐藏 | **可编辑** | **开放（带截止日）** | 蓝色 |
| `invoice_overdue` | 隐藏 | **锁定** | **开放（逾期提示）** | 橙色 |
| `invoice_submitted` | 隐藏 | 锁定(只读) | 隐藏 | 黄色 |
| `invoice_rejected` | 隐藏 | 锁定(只读) | **重新开放** | 红色 |
| `confirmed` | 隐藏 | 锁定(只读) | 隐藏 | 绿色 |

**关键限制规则：**

| 规则 | 说明 |
|------|------|
| 前置条件 | 已选择参与方式 + 已绑定该分会 |
| 两阶段审核 | 凭证初审 → 发票终审，两阶段独立审核 |
| 发票必传 | 所有会议缴费须上传发票，宽限期7工作日 |
| 单次缴费 | 一次只能缴纳一个会议的注册费 |
| 截止日期 | 超时不可提交，按钮置灰 |
| 独立审核 | 每个会议独立审核，互不影响 |

### 14.6 状态流转总图

#### 会员完整生命周期（两阶段）

```
用户注册
  → 首次登录 → 选择参与方式（非会员 / 正式会员）
  → [正式会员路径]
  → 线下转账
  → 阶段一：上传缴费凭证（OCR识别）
  → 财务初审
      ├── 通过 → 可绑定分会 + 7工作日发票截止日
      │         → 阶段二：上传电子发票（OCR识别+比对，必传）
      │         → 财务终审
      │             ├── 通过 → 正式会员（有效期1年）
      │             │         → 可自由绑定/解绑任意分会
      │             │         → 可参加已绑定分会发布的会议
      │             │         → 到期 → 自动失效
      │             │            → 所有绑定解除
      │             │            → 会议资格分级处理
      │             └── 驳回 → 重新上传发票
      └── 驳回 → 重新上传凭证
  → [非会员路径]
  → 直接使用系统 → 绑定分会 → 报名会议（非会员价）
  → 后续可随时升级为正式会员
```

```

  → 用户查看会议详情
  → 选择身份，转账（非会员按非会员价）
  → 阶段一：上传缴费凭证（OCR识别）
  → 财务初审
      ├── 通过 → 可填写参会信息 + 7工作日发票截止日
      │         → 填写参会信息

      │         → 财务终审

      │             └── 驳回 → 重新上传发票
      └── 驳回 → 重新上传凭证
  → 参会信息 + 摘要可修改至截止日期
  → 会议结束后归入历史
```

### 14.7 会员到期后会议资格分级处理

**核心原则：** "已锁定的不追溯，未锁定的清理"。

| 会员到期时的会议状态 | 处理方案 | 业务理由 |
|---------------------|---------|---------|
| `confirmed`（已终审通过） | **资格保留** | 终审已通过，资格不应被追溯取消 |
| `invoice_submitted`（待发票终审） | **冻结**（`frozenDueToExpiry = true`） | 流程已完成一半，给用户续费+恢复的机会 |
| `voucher_submitted`（待凭证初审） | **删除** | 仅完成凭证上传，损失最小 |
| `voucher_rejected`（凭证驳回） | **删除** | 审核未通过，无保留价值 |
| `invoice_pending`（待传发票） | **删除** | 用户未完成发票上传 |
| `invoice_overdue`（发票逾期） | **删除** | 已逾期未补传 |
| `invoice_rejected`（发票驳回） | **删除** | 审核未通过 |

**续费恢复：** 被冻结的会议资格在续费+终审通过后自动解锁。分会绑定不自动恢复——用户需手动重新绑定。

```typescript
function getMembershipFee(memberType: string = "standard"): number {
  const stored = localStorage.getItem("paleo_membership_fee_config");
  if (stored) {
    const config = JSON.parse(stored);
    return config[memberType] || MEMBERSHIP_FEE_CONFIG.default[memberType];

  return MEMBERSHIP_FEE_CONFIG.default[memberType];
}
```
