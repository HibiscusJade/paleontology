### 1. 项目概述与核心原则

- **项目背景：** 建设中国古生物学会（含11个分会）统一网站及会员管理系统，解决会员服务、会议管理、党建宣传及财务审计问题。
- **网站导航（11项一级栏目）：** 首页 | 学会简介 | 组织机构 | 学会服务 | 党建文化 | 学会沿革 | 历史相册 | 会员公告 | 国际交流 | 资料下载 | 规章条例（参考 http://www.chinapsc.cn/）
- **核心设计原则：**
  - **单账号全局通用：** 一个邮箱账号，全站通用。
  - **统一会员体系：** 只设一种会员身份——中国古生物学会会员，统一收取会员费。不再按分会单独收取会员费。
  - **分会自由绑定：** 有效会员可自由绑定/解绑多个分会（0~11个），绑定后接收该分会推送的会议和资讯。
  - **权限隔离：** 仅已绑定该分会的有效会员可查看该分会专属会议和内容。
  - **单次缴费：** 会员费统一缴纳一次；会议费按每个会议单笔单次缴纳，符合审计要求。
  - **两阶段审核 ⬅️ 新增：** 缴费审核分为凭证初审和发票终审两阶段。凭证初审通过后即可填写参会信息。两阶段独立审核，互不干扰。
  - **发票必传+宽限期 ⬅️ 新增：** 电子发票为必传项。凭证初审通过后7个工作日内上传，逾期锁定资格。财务可手动延期。
  - **智能+人工审核：** 凭证/发票先OCR识别（含金额比对），再人工审核。
  - **最新版本原则：** 参会信息/摘要只保留最终版本。
  - **全流程通知：** 关键操作自动触发站内信+邮件（共16个邮件模板）。
  - **分级处理 ⬅️ 新增：** 会员到期后按会议状态分级处理——`confirmed` 保留、`invoice_submitted` 冻结、其他状态删除。

### 2. 模块一：会员服务

#### 2.1 账号与注册

- **用户端（前台）**
  - **注册：** 使用邮箱注册，填写姓名、性别、单位、身份（学生/老师/其他/职称）等信息。邮箱全局唯一且不可修改。密码需6位以上。
  - **登录：** 首页右上角提供登录入口。
  - **密码重置：** 通过注册邮箱找回密码。
- **管理端（后台）**
  - **账号管理：** 管理员（学会总管理员）可查询所有会员账号，并可执行"禁用/启用"操作。禁用后账号无法登录。
  - **操作日志：** 记录账号状态变更操作。

#### 2.2 统一会员费缴纳（两阶段审核）⬅️ 更新

- **核心逻辑：** 用户注册登录后，选择【成为会员】，缴纳**中国古生物学会统一会员费**（一次缴费），经两阶段审核（凭证初审→发票终审）后成为学会有效会员（有效期1年）。不再按分会收取会员费，"分会会费"概念取消。
- **会费配置 ⬅️ 更新：** 会员费金额**后台可配置**，支持按年度独立设置、按会员类型分别定价（普通会员/学生会员/单位会员）。原型阶段默认值：普通¥200/年、学生¥100/年、单位¥5000/年。运行时通过 `getMembershipFee(memberType)` 从 localStorage（`paleo_membership_fee_config`）读取，fallback 到默认值。
- **用户端（前台）**
  - **阶段一：凭证初审**
    1. **确认信息：** 查看会员费金额（从配置读取）、收款账户信息。
    2. **线下转账：** 用户通过银行转账至学会对公账户。
    3. **上传凭证：** 上传转账回执（JPG/PNG，≤5MB）。系统OCR识别。提交后状态变更为 `voucher_submitted`（凭证初审中）。
    4. **初审结果：** 通过→状态变更为 `invoice_pending`，设置发票上传截止日（+7工作日），可绑定分会；驳回→状态变更为 `voucher_rejected`，附原因，重新上传凭证。
  - **阶段二：发票终审 ⬅️ 新增**
    5. **上传发票（必传）：** 初审通过后解锁发票上传。上传电子发票（JPG/PNG/PDF，≤10MB），OCR识别并比对金额。截止日前上传→状态变更为 `invoice_submitted`（发票终审中）；截止日后→状态变更为 `invoice_overdue`（发票逾期），会员资格锁定但仍可上传。
    6. **终审结果：** 通过→状态变更为 `active`，有效期自审核通过日起1年；驳回→状态变更为 `invoice_rejected`，附原因，重新上传发票。
  - **发票必传+宽限期 ⬅️ 新增：** 发票为必传项。宽限期为凭证初审通过后**7个工作日**（跳过周末）。逾期后会员资格锁定，上传+终审通过后自动解锁。距截止日≤3天自动提醒。财务可手动延期。
  - **状态查看：** 在"个人中心"查看会员状态（9种：not_member / voucher_submitted / voucher_rejected / invoice_pending / invoice_overdue / invoice_submitted / invoice_rejected / active / expired）。详见 §8.2 完整状态枚举。
- **管理端（后台）**
  - **会费配置：** 学会总管理员可按年度+类型设置会员费金额。修改后实时生效，变更记录操作日志。配置格式见 §8.1。
  - **审核队列分离 ⬅️ 更新：** 区分**初审队列**（`voucher_submitted`，审核凭证真实性）和**终审队列**（`invoice_submitted`，审核发票真实性+OCR比对结果审查）。两个队列独立操作，支持多选/全选+批量审核。
  - **初审操作：** 通过→状态→`invoice_pending`，设置7工作日发票截止日，通知用户；驳回→状态→`voucher_rejected`，填写原因。
  - **终审操作：** 通过→状态→`active`，有效期自审核通过日起1年；驳回→状态→`invoice_rejected`，填写原因。
  - **手动延期：** 财务可为 `invoice_pending`/`invoice_overdue` 记录设置 `invoiceExtendedDeadline`，填写延期理由。延期记录计入操作日志。
  - **手动开通/解锁：** 总管理员可为特定用户手动开通会员资格，或为逾期用户手动解除锁定。

#### 2.3 分会绑定与管理

- **核心逻辑：** 成为有效会员后，用户可进入"我的分会"页面，自由【绑定】/【取消绑定】任意分会。绑定/解绑立即生效，无需审核。
- **用户端（前台）**
  - **"我的分会"页面：**
    - 顶部状态提示：有效会员显示引导绑定文字；非会员/过期显示"请先缴纳会员费"。
    - 分会卡片网格（11个分会），每张卡片显示：名称、Logo、简介、绑定状态开关。
    - 已绑定的分会 → 绿色"已绑定" + 【取消绑定】按钮。
    - 未绑定的分会 → 灰色"未绑定" + 【绑定】按钮。
    - 可绑定0~11个分会，可随时变更。
  - **绑定条件：** 用户必须是有效会员（会员费已缴且在有效期内）。非会员或过期会员无法绑定任何分会。
  - **绑定后的效果：**
    - 首页/"会议中心"只显示已绑定分会发布的会议和文章。
    - 系统推送（站内通知+邮件）仅针对已绑定分会。
    - 未绑定分会的会议、文章完全不可见。
  - **会员到期后的处理 ⬅️ 新增：** 会员到期后所有分会绑定自动解除（`boundBranches = []`）。续费后**不自动恢复**绑定——用户需手动重新选择（用户可能在到期期间改变了研究兴趣或分会偏好）。
- **管理端（后台）**
  - **绑定统计：** 可查看各分会绑定人数、绑定分布趋势（0个/1~3个/4~6个/7~11个分会的人数分布）。
  - **无审核操作：** 绑定/解绑由用户自助完成，后台不干预。

#### 2.4 个人中心与会员状态

- **用户端（前台）**
  - **信息管理：** 可修改除邮箱外的个人信息，修改密码需验证原密码。
  - **我的分会：** 展示分会绑定开关列表，已绑定分会清晰标注，可一键解绑/重新绑定。
  - **我的会议：** 展示已绑定分会下可报名的会议及状态。⬅️ 更新：8种会议状态标签（未缴费/凭证初审中/凭证被驳回/待上传发票+截止日倒计时/发票逾期/发票终审中/发票被驳回/已确认）。`invoice_pending` 状态时显示发票截止日倒计时（X天X小时）。
  - **缴费记录：** 分为"会员费记录"和"会议费记录"两个标签页，展示历史缴费明细。⬅️ 更新：区分凭证提交时间和发票提交时间；驳回记录区分凭证驳回/发票驳回阶段（`rejectPhase` 字段）。
  - **消息中心：** 接收所有站内通知。
- **管理端（后台）**
  - **会员查询：** 管理员可按邮箱、姓名、会员状态等维度查询会员，可查看会员绑定了哪些分会。⬅️ 更新：会员状态扩展为9种。

### 3. 模块二：会议服务

#### 3.1 会议生命周期管理

- **用户端（前台）**
  - **会议列表：** 用户仅能看到**已绑定分会**发布的会议。未绑定分会的会议完全不可见。
  - **会议详情：** 展示会议通知详情、时间、地点、注册费标准。**附件下载**（通知PDF、摘要模板）需用户是该分会有效会员 + 已绑定该分会。
  - **会议状态：** 页面直观展示会议状态（报名中/进行中/已截止/已结束）。
- **管理端（后台）**
  - **会议发布：** 分会管理员可创建会议，填写详细内容（富文本），上传通知PDF和摘要模板。
  - **参数配置：** 设置学生/教师/嘉宾三类注册费标准、缴费截止日期、摘要修改截止日期。
  - **分会场配置：** 可动态添加/编辑学术分会场选项（如"01专场：古脊椎动物学"）。
  - **权限推送：** 会议发布后，仅对已绑定该分会且为有效会员的用户可见和推送通知。

#### 3.2 会议注册费缴纳（两阶段审核）⬅️ 更新

- **用户端（前台）**
  - **前置条件：** 
    1. 用户是有效会员（会员费未过期）
    2. 用户已绑定该分会
    3. 当前时间未超过缴费截止日期
  - **阶段一：凭证初审**
    1. **选择身份：** 选择学生/教师/嘉宾，系统自动计算金额。
    2. **线下转账：** 向会议指定账户转账。
    3. **上传凭证：** 上传转账回执（JPG/PNG，≤5MB）。提交后状态变更为 `voucher_submitted`。
    4. **初审结果：** 通过→状态变更为 `invoice_pending`，**解锁参会信息填写**，设置发票截止日（+7工作日）；驳回→状态变更为 `voucher_rejected`，附原因。
  - **阶段二：发票终审 ⬅️ 新增**
    5. **上传发票（必传）：** 初审通过后上传电子发票（JPG/PNG/PDF，≤10MB）。OCR识别金额+与凭证金额比对。一致→状态变更为 `invoice_submitted`；不一致→提示重新上传。截止日后→状态变更为 `invoice_overdue`，参会信息锁定但仍可上传。
    6. **终审结果：** 通过→状态变更为 `confirmed`（报名完成）；驳回→状态变更为 `invoice_rejected`。
  - **宽限期规则 ⬅️ 新增：** 发票必传，宽限期7个工作日。逾期后参会信息锁定，仍可上传。距截止日≤3天自动提醒。财务可手动延期。截止日计算：凭证初审通过日 + 7个工作日（跳过周六日）。
  - **状态→UI 对照表 ⬅️ 新增：** 各状态下前端按钮和表单的可见/可编辑状态：

| 状态 | 缴费按钮 | 参会信息 | 发票上传 | 状态标签颜色 |
|------|---------|---------|---------|------------|
| `unpaid` | 【立即缴纳注册费报名】| 隐藏 | 隐藏 | 灰色 |
| `voucher_submitted` | 隐藏 | 锁定 | 隐藏 | 黄色 |
| `voucher_rejected` | 【重新上传凭证】| 隐藏 | 隐藏 | 红色 |
| `invoice_pending` | 隐藏 | **可编辑** ✅ | **开放（带截止日）** | 蓝色 |
| `invoice_overdue` | 隐藏 | **锁定** | **开放（逾期提示）** | 橙色 |
| `invoice_submitted` | 隐藏 | 锁定(只读) | 隐藏 | 黄色 |
| `invoice_rejected` | 隐藏 | 锁定(只读) | **重新开放** | 红色 |
| `confirmed` | 隐藏 | 锁定(只读) | 隐藏 | 绿色 |

  - **多会议处理：** 用户可同时参加多个分会、多个会议，每个会议独立缴费、独立审核。一次只能提交一个会议的缴费申请，不支持合并支付。缴费记录按会议分开展示，状态互不影响。
- **管理端（后台）**
  - **审核队列分离 ⬅️ 更新：** 区分**初审队列**（审核凭证）和**终审队列**（审核发票+OCR比对结果），两个队列独立操作，支持批量审核。
  - **初审操作：** 通过→`invoice_pending`+设置发票截止日+通知；驳回→`voucher_rejected`+原因。
  - **终审操作：** 通过→`confirmed`，获得参会资格；驳回→`invoice_rejected`+原因。
  - **手动延期：** 财务可为 `invoice_pending`/`invoice_overdue` 记录延长发票截止日，需填写延期理由。
  - **每个会议独立审核，互不影响。**

#### 3.3 参会信息与摘要提交

- **用户端（前台）**
  - **前置条件 ⬅️ 更新：** 该会议的注册费**凭证初审通过**（状态为 `invoice_pending`）。即：凭证初审通过后即可填写参会信息，不必等到发票终审通过。这大幅提前了参会信息填写时机（原为终审通过后）。
  - **逾期锁定 ⬅️ 新增：** 发票逾期（`invoice_overdue`）时参会信息表单锁定（已填数据保留），上传发票进入终审后恢复正常。
  - **提交表单：**
    - **基础信息：** 姓名、性别、单位自动读取，身份（教师/学生/嘉宾）与缴费时联动。
    - **住宿统计：** 选择单间/双人间/自行安排。
    - **学术参与：** 选择口头报告/展板报告/仅参会。若选择报告，需填写报告标题。
    - **摘要上传：** 上传 .doc/.docx 文件 (≤10MB)。可随时替换，系统仅保留最新版本。
  - **修改限制：** 在管理员设置的"摘要修改截止日期"前，可无限次修改参会信息和摘要。
- **管理端（后台）**
  - **截止日期设置：** 为每个会议独立设置"参会信息/摘要修改截止日期"，超时后前端提交按钮将置灰。
  - **数据查看与导出：**
    - **参会统计：** 实时统计总参会人数、报告人数、口头/展板报告数量。
    - **住宿统计：** 交叉统计（男单间×N，男双人间×N，女单间×N，女双人间×N）。
    - **摘要导出：** 可按分会场顺序，批量导出所有参会者的最新摘要文件。
  - **特殊操作：** 管理员可后台强制修改某用户的参会信息或替换其摘要（用于处理截止日期后的特殊情况）。

#### 3.4 会员到期后会议资格分级处理 ⬅️ 新增

- **核心原则：** "已锁定的不追溯，未锁定的清理"。

| 会员到期时的会议状态 | 处理方案 | 业务理由 |
|---------------------|---------|---------|
| `confirmed`（已终审通过） | **资格保留** | 终审已通过，资格不应被追溯取消 |
| `invoice_submitted`（待发票终审） | **冻结**（`frozenDueToExpiry = true`） | 流程已完成一半，给用户续费 + 恢复的机会 |
| `voucher_submitted`（待凭证初审） | **删除** | 仅完成凭证上传，损失最小 |
| `voucher_rejected`（凭证驳回） | **删除** | 审核未通过，无保留价值 |
| `invoice_pending`（待传发票） | **删除** | 用户未完成发票上传 |
| `invoice_overdue`（发票逾期） | **删除** | 已逾期未补传 |
| `invoice_rejected`（发票驳回） | **删除** | 审核未通过 |

- **实现逻辑：** 会员到期时遍历所有 `conferenceRegs`，按上表规则分级处理，清空 `boundBranches = []`，设置 `societyMembership.status = "expired"`。
- **续费恢复：** 用户续费 + 发票终审通过后，自动检测所有 `frozenDueToExpiry = true` 的会议，批量解锁。分会绑定**不自动恢复**——用户需手动重新绑定。

| `invoice_pending` | 待上传发票 | 蓝色 + 倒计时 | 凭证已通过，请于 X 日前上传发票；可填参会信息 |
| `invoice_overdue` | 发票逾期 | 橙色 | 发票未在截止日前上传，请尽快补传；参会信息锁定 |
| `invoice_submitted` | 发票终审中 | 黄色 | 发票已上传（OCR 比对通过），等待财务终审 |
| `invoice_rejected` | 发票被驳回 | 红色 + 原因 | 发票审核未通过，需重新上传 |
| `confirmed` | ✓ 已确认 | 绿色 | 终审通过，报名已完成 |

#### 8.7 关键术语表

| 术语 | 定义 |
|------|------|
| 凭证 | 银行汇款/转账的电子回单截图，证明"已付款" |
| 发票 | 学会开具的电子发票，证明"合规" |
| 初审 | 财务审核凭证的阶段（阶段一） |
| 终审 | 财务审核发票的阶段（阶段二） |
| 宽限期 | 凭证初审通过后，上传发票的 7 个工作日时限 |
| 工作日 | 周一至周五（不含周末和法定假日） |
| 冻结 | 会员到期后对 `invoice_submitted` 会议资格的处理标记（`frozenDueToExpiry`） |
| OCR | 光学字符识别，自动提取发票/凭证上的金额信息并进行比对 |
| 兜底方案 | 后台手动延期/解锁等管理员操作 |
| 截止日倒计时 | `invoice_pending` 状态下实时显示距离截止日的剩余时间 |
