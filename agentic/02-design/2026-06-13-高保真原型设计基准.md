## 使用结论

**官网 V3** 是中国古生物学会官网的高保真设计产物。后续官网开发、交互还原、页面验收和视觉走查，应优先参考该文件。

**管理后台 V1** 是当前最新的运营端高保真设计产物。后续后台开发、审核流程还原、权限界面和数据统计页面验收，应优先参考该文件。

为避免桌面文件移动或被覆盖，已将当前版本复制到仓库 `agentic/02-design/references/` 下作为设计快照。后续如果桌面 HTML 有新版本，应按日期复制新快照，并更新本文件中的哈希和版本说明。

简单说：**界面长什么样、怎么点，看高保真；业务怎么算、谁能操作，看 PRD。**

## 主要可见业务状态

### 官网

高保真原型中已出现以下状态或提示，可作为前端状态文案参考：

- `未缴费`
- `凭证初审中`
- `凭证被驳回`
- `待上传发票`
- `发票逾期`
- `发票终审中`
- `发票被驳回`
- `已确认`
- `凭证初审已通过`
- `发票终审已通过`
- `请先等待凭证初审通过后再上传发票。`
- `请重新上传清晰的缴费凭证。`
- `电子发票提交成功，请等待财务终审。`

### 管理后台

高保真原型中已出现以下状态或操作，可作为后台状态机与操作文案参考：

- `待初审` / `初审已通过` / `初审已驳回`
- `待终审` / `终审已通过` / `终审已驳回`
- `待上传发票` / `发票逾期`
- `已确认` / `已发布` / `草稿`
- `已手动开通会员` / `已批量驳回`
- `驳回凭证` / `驳回发票`（需填写驳回原因并通知用户）
- `确认延期`（延长用户上传发票截止日）
- `分会已启用` / `分会已禁用`
- `您没有权限访问该页面`

这些文案可以直接作为页面初稿参考，但最终状态机仍应在技术设计中统一定义。

## 设计还原要求

- 开发时应尽量还原高保真中的导航结构、模块层级、颜色气质、卡片样式、按钮形态、表格/表单密度和状态提示。
- 高保真中的交互路径应作为验收依据之一，包括登录、会员服务、分会绑定、会议报名、缴费审核、参会信息等官网路径，以及审核工作台、用户管理、分会管理、会议管理等后台路径。
- 若业务规则已在 PRD 中更新，但高保真尚未同步，应在开发前标注差异并确认处理方式。
- 若高保真中存在演示数据、模拟会议、模拟会员状态，开发时应替换为真实接口数据或后台配置数据。
- 高保真 HTML 是打包产物，不建议直接二次开发；开发时应在工程技术栈中重建组件和页面。
