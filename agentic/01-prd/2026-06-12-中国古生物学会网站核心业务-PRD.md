## 一、需求目标

建设中国古生物学会官方网站及会员服务系统，优先完成会员管理、会员费缴纳、会议注册费缴纳、参会信息提交和党建文化展示能力，满足秘书处日常管理、财务审计、分会会议组织和网站内容维护需求。

## 二、核心原则

| 原则 | 说明 |
|---|---|
| 单账号全站通用 | 用户使用一个账号登录网站，后续通过身份和状态区分权限 |
| 统一会员体系 | 会员统一为中国古生物学会会员，会员费只收一次 |
| 分会自由绑定 | 会员可自由绑定/解绑总学会及 11 个分会，绑定关系用于会议和文章推送 |
| 会议一场一缴 | 会议注册费按会议单独缴纳，一次只能提交一个会议的缴费申请 |
| 非会员可参会 | 在现有会员流程基础上，新增非会员会议注册缴费能力 |
| 线下转账为主 | 当前阶段不做线上收费，继续采用线下转账、上传凭证、智能识别、人工审核 |
| 提交可追溯 | 缴费、审核、驳回、绑定、导出、后台修改等关键动作应记录日志 |
| 工期优先 | 不主动增加超出当前客户要求的复杂能力，先保证核心流程交付 |

## 四、用户角色

| 角色 | 说明 | 主要能力 |
|---|---|---|
| 普通用户/非会员 | 注册账号但不缴纳会员费，可按非会员价格参加会议 | 浏览公开内容、绑定会议相关分会、缴纳非会员会议注册费、提交参会信息 |

| 缴费会员/有效会员 | 会员费审核通过且在有效期内 | 正常使用会员服务和会议服务 |

- 分会绑定/解绑。
- 非会员会议注册缴费能力。
- 会议发布、会议注册费缴纳。
- 参会信息、报告类型、摘要文件、住宿信息提交。

## 六、会员服务

### 6.1 会员注册

- 用户通过网站注册账号。
- 用户可选择进入会员路线，申请成为中国古生物学会会员。
- 会员注册不按分会拆分，所有会员统一归属中国古生物学会。
- 会员费只缴纳一次，不因绑定多个分会重复收费。

### 6.2 会员费缴纳

- 当前阶段采用线下转账。
- 用户上传转账凭证和发票材料。
- 系统可通过智能识别辅助提取金额、账号、时间等信息。
- 财务审核员进行人工审核。
- 审核通过后，会员进入有效状态，有效期一年。
- 每次会员费只支持缴纳一年，提前续费时在原有效期基础上顺延。

### 6.3 会员状态

| 状态 | 说明 |
|---|---|
| 未入会 | 已注册账号，但未申请或未完成会员费缴纳 |
| 审核中 | 已提交缴费材料，等待审核 |
| 已驳回 | 缴费材料未通过，需要按驳回原因重新提交 |
| 有效会员 | 会员费审核通过且在有效期内 |
| 已过期 | 会员有效期届满，需要续费 |

具体审核状态可在技术设计阶段细分为凭证初审、发票终审等子状态。

## 七、分会绑定

- 用户可绑定总学会及 11 个分会。
- 会员可绑定一个、多个或全部分会。
- 绑定和取消绑定应即时生效。
- 绑定关系用于接收对应学会/分会的会议通知和文章推送。
- 未绑定的学会/分会，不推送对应会议和文章。
- 建议记录绑定/解绑日志，包括操作人、分会、操作时间和操作类型。
- 管理界面中应新增并置顶“中国古生物学会（总学会）”模块。
- 原“专业分会绑定管理”建议调整为“会议绑定管理”。

## 八、会议服务

### 8.1 会议发布

- 总学会和 11 个分会均可发布会议。
- 12 个会议分别发布、分别配置缴费周期。
- 每个会议可配置：
  - 会议名称。
  - 所属学会/分会。
  - 会议时间、地点。
  - 会议通知 PDF。
  - 摘要模板。
  - 缴费截止日期。
  - 摘要提交/修改截止日期。
  - 会员会议注册费。
  - 非会员会议注册费。

### 8.2 会议注册费缴纳

- 会议注册费按会议单独缴纳。
- 用户参加几个会议，就缴纳几次会议注册费。
- 一次只能提交一个会议的缴费申请，不支持多个会议合并支付。
- 必须在单次会议缴费截止日期前提交申请，超时不可提交。
- 当前阶段采用线下转账、上传凭证、智能识别、人工审核的流程。
- 会员和非会员都可以参加会议，但会议注册费金额不同。
- 会议通知中需要明确展示会员价和非会员价。

### 8.3 非会员参会

- 非会员可以注册网站账号。
- 非会员不缴纳会员费，但可以参加会议。
- 非会员可以绑定总学会及分会，用于查看或接收相关会议通知。
- 非会员会议注册费高于会员会议注册费。
- 非会员后续参会信息、摘要、住宿、报销材料下载流程与会员一致。
- 实现上优先在现有会员流程基础上增加非会员会议注册缴费能力。
- 非会员转会员优先考虑同账号升级，不建议强制注销后重新注册；该点需甲方最终确认。

## 九、参会信息提交

### 9.1 前置条件

- 用户已完成对应会议注册费缴纳并通过审核后，进入参会信息提交阶段。

### 9.2 填写内容

| 字段 | 说明 |
|---|---|
| 参会身份 | 学生、教师、嘉宾等，具体类型由会议配置 |
| 报告类型 | 口头报告、展板报告、仅参会 |
| 报告标题 | 选择口头报告或展板报告时填写 |
| 摘要文件 | Word 文件，支持替换上传，保留最新版本 |
| 住宿选择 | 单间、双人间、自行安排；是否拆分男女单/双人间在设计阶段确认 |

### 9.3 截止与修改

- 摘要提交截止日期前，用户可修改参会信息和摘要。
- 截止日期后，前台提交和修改入口关闭。
- 后台管理员可在特殊情况下强制修改参会信息或摘要，但必须记录操作日志和修改原因。

### 9.4 学生身份验证

- 当前阶段不主动增加学生证上传、学信网校验或外部 API。
- 默认由用户线上自行选择学生身份。
- 线下会议签到时由会务人员人工查验学生证。
- 如甲方后续明确要求线上学生身份审核，再作为补充需求。

### 9.5 会员过期后的会议资格

- 已终审通过、报名已确认的会议资格，即使会员后来到期，也不取消，照常参会。
- 发票已提交但尚未终审时，若会员过期，则会议资格暂时冻结；待会员续费并终审通过后恢复。
- 其他早期状态可在续费后重新报名或继续流程，具体状态机在技术设计阶段细化。

## 十、缴费提交与驳回

- 用户提交缴费材料后，系统保留最终提交信息。
- 验证通过后，用户不能自行返回修改。
- 如需重新提交，需要秘书处或管理员手动驳回。
- 驳回后，用户才能重新上传凭证和发票。
- 驳回必须填写原因，并保留历史记录。
- 退款等特殊场景通过人工驳回和线下处理完成。

## 十一、后台统计

后台需要支持以下统计和导出：

- 当年会员数、已缴会员数、会员费金额。
- 学生/教师等身份人数和金额。
- 各分会绑定人数。
- 各会议报名人数、已缴费人数、会议费金额。
- 口头报告数量、展板报告数量。
- 住宿统计。
- 缴费记录、审核记录、参会信息导出。
